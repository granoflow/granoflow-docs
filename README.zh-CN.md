## 2. 为什么建议你现在就试

你不需要先把所有文档看完。我们更建议你先用起来，再判断是否适合自己。

常见试用场景（示例）：
- 你每天打开很多工具：聊天软件里有待办，备忘录里有想法，日历里有安排，但你突然觉得，自己什么都不想做。
- 你手上有一个持续几周的项目，不是一天能做完；你每天都在加班，但每天总有任务逾期，你感觉非常焦虑。
- 你疲于应付突发事件，总是被外部干扰打断，回到任务时已经忘了刚才做到哪里。
- 你不是想把每天塞满，而是想在工作、学习、健康、创作之间保持一个能长期回来的节奏。
- 你愿意试用一个仍在快速迭代的产品，并把真实问题、卡点和改进建议直接反馈给我们。

> 产品功能细节与完整说明，已在手册前几章节详细展开；本 README 不重复介绍。

## 3. 试用激励计划（送年费会员）

我们希望把真实用户反馈放在产品迭代中心。为此，当前提供以下激励：

### 3.1 奖励类型
- **年费会员**（按规则发放）

### 3.2 获奖方式
满足以下任一项，即可获得年费会员：

1. **发布 issue，且经确认有效**  
   - 发布前请先搜索现有 issues，确认是否已有人提交同一问题。
   - 同一问题仅奖励首个有效发布者；重复 issue 不重复发放奖励。
   - 例如：可复现的 Bug、明确的异常行为、可定位的问题线索。

2. **提出需求建议，且被采纳**  
   - 发布前请先搜索现有 issues，确认是否已有人提交同类需求。
   - 同一需求方向仅奖励首个被采纳的有效建议；重复建议不重复发放奖励。
   - 例如：具备明确场景、目标与价值的功能建议。

3. **提交高质量体验分享，被确认“言之有物”**  
   - 发布位置要求（满足其一）：
     - 无需登录即可公开访问的平台；或
     - 全球主流社交媒体平台（如 X/Twitter、YouTube、Reddit、Facebook、Instagram、LinkedIn、TikTok、知乎、微博等）
   - 必须提供原帖直链，且发布满 24 小时后链接仍可访问（未删帖、未转私密）。
   - 内容要求：
     - 以个人工作/生活/实践经验分享为主，不写成推销文案；
     - 在真实叙事中顺带提到 Granoflow 在某一个具体点上的帮助；
     - 至少包含“问题 → 做法 → 结果”的最小闭环。

### 3.3 判定与发放说明
- “有效”“采纳”“言之有物”由维护团队基于内容质量与实际价值判定。
- 同一问题重复提交、缺少关键信息、明显灌水内容，不计入奖励。
- issue 奖励采用“首发有效”原则：同一问题只奖励第一个有效发布者。
- 需求建议奖励采用“首发采纳”原则：同一需求方向只奖励第一个被采纳的有效建议。
- 具体发放流程与时效，以后续公开规则为准（将在仓库与相关渠道同步）。

### 3.4 发布入口
- Issue 问题反馈与需求建议：GitHub Issues  
  <https://github.com/granoflow/granoflow-docs/issues>
- 体验分享：发布到满足 3.2 规则的可公开核验平台。

## 5. 我们最希望收到的反馈类型

为了提高处理效率，也提高你获得奖励的成功率，建议反馈尽量包含：

### 5.1 Bug 反馈
- 发生环境（设备/系统/版本）
- 复现步骤
- 预期结果 vs 实际结果
- 截图或录屏（如有）

### 5.2 需求建议
- 你要解决的具体场景
- 当前替代方案及其不足
- 你期望的结果（最好可衡量）

### 5.3 体验分享反馈
- 先分享你的真实工作/生活/实践经验（非营销文案）
- 说明 Granoflow 在其中一个具体点上提供了什么帮助
- 用“问题 → 做法 → 结果”描述变化（结果尽量具体）
- 提供原帖直链，并确保 24 小时后仍可访问

## 6. 透明处理机制

- Granoflow 不是开源软件，但核心功能设计上基本可以免费使用；付费会员主要用于支持持续开发，并解锁更完整的扩展能力。
- 我们会持续公开反馈处理进展（如状态、采纳情况、后续计划）。
- 奖励发放将遵循可追踪原则，同时注意隐私保护。
- 活动规则如有更新，会在公开渠道同步说明。

## 8. FAQ（待持续补充）

- 谁可以参加激励计划？
- 一个人可以多次获得奖励吗？
- 提交后一般多久能收到确认？
- 年费会员通过什么方式发放？
- 什么样的“体验分享”会被判定为有效？
  - 内容以个人工作/生活/实践经验为主，非营销推广文案。
  - 清楚包含“问题 → 做法 → 结果”的最小闭环，并有具体细节。

- 哪些“体验分享”会被判定为无效？
  - 纯广告式文案、口号式安利、缺少个人真实情境。
  - 内容空泛，无法判断具体问题、做法与结果。
  - 仅提供主页/合集链接，未提供可核验的原帖直链。
