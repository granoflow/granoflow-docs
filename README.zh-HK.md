## 3. 試用激勵計劃（送年費會員）

我哋想將真實用戶回饋擺喺產品迭代最核心位置，所以而家有以下激勵：

### 3.1 獎勵類型
- **年費會員**（按規則發放）

### 3.2 獲獎方式
符合以下其中一項，就可以拎到年費會員：

1. **發布 issue，並經確認有效**  
   - 發布前請先搜尋現有 issues，睇下有冇人已經報過同一問題。
   - 同一問題只獎勵首個有效發布者；重複 issue 唔會重複發獎。
   - 例如：可重現嘅 Bug、明確異常行為、可定位嘅問題線索。

2. **提出需求建議，並被採納**  
   - 發布前請先搜尋現有 issues，睇下有冇同類需求。
   - 同一需求方向只獎勵首個被採納嘅有效建議；重複建議唔會重複發獎。
   - 例如：有清楚場景、目標同價值嘅功能建議。

3. **提交高質體驗分享，並被確認「言之有物」**  
   - 發布位置要求（符合其一）：
     - 毋須登入都可公開查看嘅平台；或
     - 全球主流社交媒體平台（例如 X/Twitter、YouTube、Reddit、Facebook、Instagram、LinkedIn、TikTok、知乎、微博等）
   - 必須提供原帖直連，而且發布滿 24 小時後條 link 仍然睇到（未刪帖、未轉私密）。
   - 內容要求：
     - 以個人工作/生活/實踐經驗分享為主，唔好寫成推銷文案；
     - 喺真實敘事入面，自然提到 Granoflow 喺某一個具體位點樣幫到你；
     - 至少包含「問題 → 做法 → 結果」最小閉環。

### 3.3 判定與發放說明
- 「有效」「採納」「言之有物」由維護團隊按內容質量同實際價值判定。
- 同一問題重複提交、缺少關鍵資料、明顯灌水內容，都唔計入獎勵。
- issue 獎勵採用「首發有效」原則：同一問題只獎勵第一個有效發布者。
- 需求建議獎勵採用「首發採納」原則：同一需求方向只獎勵第一個被採納嘅有效建議。
- 具體發放流程同時效，以後續公開規則為準（會喺倉庫同相關渠道同步）。

### 3.4 發布入口
- Issue 問題回饋同需求建議：GitHub Issues  
  <https://github.com/granoflow/granoflow-docs/issues>
- 體驗分享：發布到符合 3.2 規則、可公開核驗嘅平台。

## 5. 我哋最想收到嘅回饋類型

為咗提高處理效率，同時提高你拎到獎勵嘅成功率，建議回饋盡量包含：

### 5.1 Bug 回饋
- 發生環境（設備/系統/版本）
- 重現步驟
- 預期結果 vs 實際結果
- 截圖或錄屏（如有）

### 5.2 需求建議
- 你想解決嘅具體場景
- 現有替代方案同其不足
- 你期望嘅結果（最好可衡量）

### 5.3 體驗分享回饋
- 先講你真實嘅工作/生活/實踐經驗（非行銷文案）
- 說明 Granoflow 喺其中一個具體位點樣幫到你
- 用「問題 → 做法 → 結果」描述變化（結果愈具體愈好）
- 提供原帖直連，並確保 24 小時後仍可訪問

## 8. FAQ（持續補充）

- 邊個可以參加激勵計劃？
- 一個人可唔可以多次拎獎勵？
- 提交後一般要等幾耐先有確認？
- 年費會員會點樣發放？
- 咩情況先算有效「體驗分享」？
  - 內容以個人工作/生活/實踐經驗為主，唔係行銷推廣文案。
  - 清楚包含「問題 → 做法 → 結果」最小閉環，並有具體細節。
  - 喺敘事中自然提到 Granoflow 喺某個具體位點樣幫到你。
  - 提供原帖直連，而且發布滿 24 小時後仍可訪問。
- 邊啲「體驗分享」會被判定為無效？
  - 純廣告式文案、口號式安利、缺少個人真實情境。
  - 內容太空泛，無法判斷具體問題、做法同結果。
  - 只提供主頁/合集連結，未提供可核驗嘅原帖直連。
  - 24 小時內刪帖、轉私密，或連結無法外部訪問核驗。
