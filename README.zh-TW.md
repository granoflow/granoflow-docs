你不需要先把所有文件看完。我們更建議你先用起來，再判斷是否適合自己。

- 你正被重複流程、資訊分散、上下文切換打斷。
- 你希望把關鍵流程變得更清楚、更可追蹤。

## 3. 試用激勵計畫（送年費會員）

我們希望把真實使用者回饋放在產品迭代中心。為此，當前提供以下激勵：

### 3.1 獎勵類型
- **年費會員**（依規則發放）

### 3.2 獲獎方式
滿足以下任一項，即可獲得年費會員：

1. **發布 issue，且經確認有效**  
   - 發布前請先搜尋現有 issues，確認是否已有人提交同一問題。
   - 同一問題僅獎勵首個有效發布者；重複 issue 不重複發放獎勵。
   - 例如：可重現的 Bug、明確的異常行為、可定位的問題線索。

2. **提出需求建議，且被採納**  
   - 發布前請先搜尋現有 issues，確認是否已有人提交同類需求。
   - 同一需求方向僅獎勵首個被採納的有效建議；重複建議不重複發放獎勵。
   - 例如：具備明確情境、目標與價值的功能建議。

3. **提交高品質體驗分享，被確認「言之有物」**  
   - 發布位置要求（滿足其一）：
     - 無需登入即可公開瀏覽的平台；或
     - 全球主流社群媒體平台（如 X/Twitter、YouTube、Reddit、Facebook、Instagram、LinkedIn、TikTok、知乎、微博等）
   - 必須提供原帖直連，且發布滿 24 小時後連結仍可瀏覽（未刪帖、未轉私密）。
   - 內容要求：
     - 以個人工作/生活/實踐經驗分享為主，不寫成推銷文案；
     - 在真實敘事中順帶提到 Granoflow 在某一個具體點上的幫助；
     - 至少包含「問題 → 做法 → 結果」的最小閉環。

### 3.3 判定與發放說明
- 「有效」「採納」「言之有物」由維護團隊基於內容品質與實際價值判定。
- 同一問題重複提交、缺少關鍵資訊、明顯灌水內容，不計入獎勵。
- issue 獎勵採用「首發有效」原則：同一問題只獎勵第一個有效發布者。
- 需求建議獎勵採用「首發採納」原則：同一需求方向只獎勵第一個被採納的有效建議。
- 具體發放流程與時效，以後續公開規則為準（將在倉庫與相關管道同步）。

### 3.4 發布入口
- Issue 問題回饋與需求建議：GitHub Issues  
  <https://github.com/granoflow/granoflow-docs/issues>
- 體驗分享：發布到符合 3.2 規則的可公開核驗平台。

## 4. 如何參與（建議路徑）

1. **先試用 Granoflow**（依你的真實情境使用）
2. **提交回饋**（Issue / 需求建議 / 體驗分享）

   > **Issue/需求建議入口：**  
   > <https://github.com/granoflow/granoflow-docs/issues>

3. **等待確認結果**（是否有效/採納）
4. **領取年費會員**（符合條件即可）

## 5. 我們最希望收到的回饋類型

為了提升處理效率，也提高你獲得獎勵的成功率，建議回饋盡量包含：

### 5.1 Bug 回饋
- 發生環境（裝置/系統/版本）
- 重現步驟
- 預期結果 vs 實際結果
- 截圖或錄影（如有）

### 5.2 需求建議
- 你要解決的具體情境
- 目前替代方案及其不足
- 你期望的結果（最好可衡量）

### 5.3 體驗分享回饋
- 先分享你的真實工作/生活/實踐經驗（非行銷文案）
- 說明 Granoflow 在其中一個具體點上提供了什麼幫助
- 用「問題 → 做法 → 結果」描述變化（結果盡量具體）
- 提供原帖直連，並確保 24 小時後仍可瀏覽

## 8. FAQ（持續補充）

- 誰可以參加激勵計畫？
- 一個人可以多次獲得獎勵嗎？
- 提交後一般多久能收到確認？
- 年費會員透過什麼方式發放？
- 什麼樣的「體驗分享」會被判定為有效？
  - 內容以個人工作/生活/實踐經驗為主，非行銷推廣文案。
  - 清楚包含「問題 → 做法 → 結果」的最小閉環，並有具體細節。
  - 在敘事中自然提到 Granoflow 在某個具體點上的幫助。
  - 提供原帖直連，且發布滿 24 小時後仍可瀏覽。
- 哪些「體驗分享」會被判定為無效？
  - 純廣告式文案、口號式安利、缺少個人真實情境。
  - 內容空泛，無法判斷具體問題、做法與結果。
  - 僅提供主頁/合集連結，未提供可核驗的原帖直連。
  - 24 小時內刪帖、轉私密，或連結無法外部訪問核驗。

## 9. 倉庫邊界說明

本倉庫僅存放 **Granoflow 對外公開文件**（文本與公開截圖等）。

以下內容不在本倉庫維護範圍內：
- 網站實作檔案
- Astro / Starlight 設定
- 自訂 CSS 與客戶端腳本
- 建置與部署腳本
- 私有筆記與密鑰資訊
