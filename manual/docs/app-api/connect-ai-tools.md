Granoflow 可以让 Codex、OpenClaw、Hermes 这类 AI 工具按接口服务连接当前设备。桌面端通常是同一台电脑上的 AI 工具连接 `127.0.0.1`；手机端更常见的是电脑上的 AI 工具通过同一 Wi-Fi 下的手机局域网地址连接这台手机。手机端细节见 [手机端局域网连接](/manual/app-api/mobile-lan-connection/)。

## 推荐方式

在“本机接口服务”页面点击“查看连接方式”。这个按钮会复用顶部 AI 助手，不会新建另一套助手。它会把当前页面、相关手册和连接意图一起交给已选 AI 工具。

AI 收到的提示会要求它优先阅读：

- App 接口总览
- 连接 Codex、OpenClaw、Hermes
- 本机地址与浏览器调试
- 官方文档调试
- 安全设置与访问码

如果手册没有写到某个工具的具体步骤，AI 应该明确说明“当前手册还没有写到这一步”，而不是编造操作。

## 连接前检查

1. 在 App 中打开“本机接口服务”。
2. 开启“开启接口服务”。
3. 确认是否需要访问码保护。
4. 如果工具运行在同一台桌面设备，优先使用页面显示的 `127.0.0.1` 地址。
5. 如果工具运行在电脑上，而 Granoflow 在手机上，使用手机页面显示的局域网地址，例如 `http://192.168.1.23:<端口>`。
6. 如果工具需要通过浏览器文档页调试，改用“官方文档调试”和临时访问码。

## 手机端连接

手机端通常不是让手机本机运行 AI 工具，而是让同一 Wi-Fi 下的电脑连接手机上的 Granoflow。Android、iPhone 和 iPad 的限制不同，先阅读 [手机端局域网连接](/manual/app-api/mobile-lan-connection/)。

## 工具差异

Codex、OpenClaw、Hermes 可能有不同的连接入口、配置文件或插件机制。Granoflow 手册只说明 Granoflow 侧需要提供什么，不承诺每个第三方工具的界面位置保持不变。

如果某个工具要求填入 API base URL，通常应使用 App 页面显示的连接地址，而不是 `https://granoflow.com/api/`。桌面端多为 `127.0.0.1` 地址；手机端多为局域网地址。如果工具要求提供接口文档，再让它阅读本组手册和公开 API 文档。

## 安全提醒

不要把访问码粘贴到不可信工具或公开聊天中。不要为了省事直接开启“允许任何设备来源”。如果你不确定某个工具是否可信，先关闭接口服务，确认后再重新开启。
