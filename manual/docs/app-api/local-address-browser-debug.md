“本机地址”是当前设备上 Granoflow 接口服务的地址。桌面端默认形式通常是 `http://127.0.0.1:<端口>`。手机端更常用的是局域网地址，例如 `http://192.168.1.23:<端口>`。手机端完整说明见 [手机端局域网连接](/manual/app-api/mobile-lan-connection/)。

## 复制地址

开启接口服务后，点击“复制地址”可以把当前连接地址放入剪贴板。桌面端通常把它填入同一台电脑上的脚本、自动化工具或 AI 工具。手机端通常把它发给同一 Wi-Fi 下的电脑端工具。

如果接口服务关闭，复制和打开操作会保持不可用。先开启“开启接口服务”，再复制地址。

## 在浏览器打开

点击“在默认浏览器打开”会打开健康检查地址，用来确认本机接口是否能响应。这个操作适合快速判断服务是否已经启动，不等同于完整业务接口测试。

如果端口被其他程序占用，页面会提示当前端口不可用。此时先修改端口，再重启或重新开启 Granoflow，并同步更新所有已经配置过旧端口的工具。

## 修改端口

端口用于决定本机接口服务监听哪个地址。修改端口后，已经连接过 Granoflow 的脚本、浏览器页面或 AI 工具也需要同步修改。

如果你只是跟随手册操作，通常不需要改端口。只有在端口冲突、组织策略要求或已有工具占用默认端口时，才需要调整。

## 常见误解

`127.0.0.1` 代表当前设备。把这个地址发给另一台设备，另一台设备会尝试连接它自己，而不是你的 Granoflow。

手机端如果要让电脑连接，应使用手机页面显示的局域网地址，而不是 `127.0.0.1`。如果页面没有显示局域网地址、电脑和手机不在同一 Wi-Fi，或当前网络隔离设备之间的访问，电脑可能无法连接。详细说明见 [手机端局域网连接](/manual/app-api/mobile-lan-connection/)。

如果需要让其他设备连接当前设备，先阅读 [安全设置与访问码](/manual/app-api/security-access-code/)，不要直接打开任何设备来源。
