手机端连接和桌面端不同。桌面端通常让同一台电脑上的 AI 工具、浏览器或脚本连接 `127.0.0.1`。手机端更常见的用法，是让同一 Wi-Fi 下的电脑连接这台手机。

## 开启和关闭

进入“本机接口服务”后，手机端页面会先显示接口服务未开启。点击“开启临时 HTTP 接口”后，页面才会启动前台连接并显示可复制的局域网地址。

连接完成或不再需要时，点击“关闭接口”。离开页面、切换到其他 App、锁屏或系统挂起时，当前连接也可能被停止。

## 为什么手机端使用局域网地址

`127.0.0.1` 只代表当前设备自己。把手机上的 `127.0.0.1` 发给电脑，电脑会尝试连接电脑自己，而不是连接手机上的 Granoflow。

手机端需要使用页面显示的局域网地址，例如：

```text
http://192.168.1.23:<端口>
```

电脑和手机必须在同一 Wi-Fi 或可互通网络中。访客 Wi-Fi、热点隔离、公司网络隔离、VPN 或系统防火墙都可能让电脑无法访问手机地址。

## Android

Android 可以显示手机的局域网地址，让同一 Wi-Fi 下的电脑、浏览器或 AI 工具连接这台 Android 设备。

当前页面只承诺用户手动开启后的前台连接。后台保持连接属于额外能力；如果未来页面提供“后台保持连接”，开启后 Android 会显示常驻通知，表示 GranoFlow 接口服务正在运行，并需要提供通知或 App 页面内的停止入口。

Android 连接仍不代表永久运行。系统省电策略、厂商后台限制、手动结束 App、网络切换或关闭接口服务，都可能让连接中断。

## iPhone 和 iPad

iPhone 和 iPad 只适合前台临时连接。请保持 GranoFlow 打开并停留在前台。

锁屏、切换到其他 App、系统挂起或系统回收后，连接可能中断。iPhone / iPad 页面不应把调试倒计时理解为后台持续运行承诺。

如果你要让电脑上的 AI 工具连接 iPhone 或 iPad，先点击“开启临时 HTTP 接口”，再使用页面显示的局域网地址，并在连接期间保持 GranoFlow 在前台。

## 和 AI 工具连接时

手机端通常不是让手机本机运行 Codex、OpenClaw 或 Hermes，而是让电脑端工具通过局域网地址连接手机上的 Granoflow。

如果 AI 工具要求填写 API base URL：

- 桌面端通常填写 `http://127.0.0.1:<端口>`。
- 手机端填写页面显示的局域网地址，例如 `http://192.168.1.23:<端口>`。

如果 AI 工具仍然建议你复制手机上的 `127.0.0.1` 给电脑使用，这个建议是错误的。

## 安全提醒

手机端显示局域网地址后，同一 Wi-Fi 下的其他设备可能尝试连接这台手机。不要为了让连接更容易成功而直接打开“允许任何设备来源”。

先开启访问码保护，确认网络可信，再决定是否让其他设备连接。完成连接后，如果不再需要，关闭接口服务。
