本机接口服务会让工具连接当前设备，因此安全设置很重要。默认情况下，你应该只开启自己真正需要的连接能力。

桌面端通常只允许同一台电脑访问。手机端如果显示局域网地址，同一 Wi-Fi 下的其他设备可能尝试连接这台手机，因此更需要访问码保护。手机端连接限制见 [手机端局域网连接](/manual/app-api/mobile-lan-connection/)。

桌面端的连接日志可用于检查最近是否有陌生地址或失败请求。日志只保存在当前设备，最多保留最近一段时间；它不是云端审计记录，也不会同步到其他设备。

## 访问码保护

访问码用于限制谁可以调用本机接口。把访问码交给工具前，先确认这个工具是否可信、是否会同步配置到云端、是否可能把配置展示给其他人。

如果工具只在你自己的设备上运行，仍建议使用访问码保护。这样即使其他页面或脚本知道本机地址，也不能直接调用接口。

## 临时访问码

“官方文档调试”使用的是临时访问码。它适合短时间验证官方文档页面和当前设备之间的连接，不适合作为长期工具配置。

调试结束后可以手动关闭；没有手动关闭时，到期后也会自动失效。

## 允许任何设备来源

“允许任何设备来源”是进阶设置。开启后，其他设备上的页面或工具可能尝试查询和操作当前设备。手机端尤其要谨慎，因为局域网地址会让同一 Wi-Fi 下的电脑或浏览器有机会连接这台手机。开启前必须先理解访问码保护。

不要为了让连接“更容易成功”而直接打开这个开关。只有在你明确知道需要跨设备访问、网络环境可信、访问码已经配置好，并且知道如何关闭时，才考虑开启。

## 如果不确定

如果你不确定某个工具需要哪种权限，先保持“允许任何设备来源”关闭。让 AI 助手阅读本组手册后解释连接原则，再决定是否继续。
