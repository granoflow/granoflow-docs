备份是你的"后悔药"。但它不会自动生效——你需要主动创建，主动保存到安全的地方。

![备份与恢复管理界面](../../screenshots/zh-cn/data-backup-restore-management.png)

## 备份 vs 同步，有什么区别

| | 备份 | 云端同步 |
| --- | --- | --- |
| 记录历史状态？ | ✅ 是某个时间点的快照 | ❌ 只有当前状态 |
| 误删后能恢复？ | ✅ 可以恢复到备份时间点 | ❌ 删除会同步到云端 |
| 需要主动操作？ | ✅ 需要手动导出 | ✅/❌ 自动进行，但不保存历史 |

## 什么时候应该做备份

- 升级 App 大版本之前
- 换手机或重装系统之前
- 删除大量任务或项目之前
- 完成一个重要阶段后想留存记录

## 怎么做备份

1. 进入 GranoFlow 设置 → 数据/备份
2. 选择导出备份
3. 把备份文件保存到你能控制的位置（iCloud、本地文件夹、电脑……）

## 怎么从备份恢复

1. 进入 GranoFlow 设置 → 数据/备份
2. 选择导入备份
3. 找到备份文件并导入

:::caution[恢复会覆盖当前数据]
从备份恢复是一个覆盖操作——导入后，当前设备的数据会被备份文件替换。如果你要保留当前设备的最新内容，请先导出当前备份，再导入旧备份。
:::
