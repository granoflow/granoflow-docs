你现在最需要做的是：在还能打开 GranoFlow 的设备上，把关键密钥保存到密码管理器或安全的地方。以后换设备、重装 App、导入本地备份，或看到“云端同步恢复”时，GranoFlow 可能会要求你输入对应密钥；没有它，某些加密数据可能打不开。

![加密与恢复密钥 界面截图](../../screenshots/zh-cn/data-encryption-recovery-key.png)

GranoFlow 的备份和云端同步都使用加密。密钥像保险箱钥匙：**没有正确的钥匙，连 GranoFlow 自己的服务器也无法读取被保护的数据**。这也意味着：**如果你自己把密钥弄丢了，GranoFlow 无法帮你重置或找回已经被它保护的数据。**

这里有两个用户会看到的名字：创建本地备份时显示的是“数据密钥”；云端同步恢复里输入的是“云端同步密码”。它们底层都服务于加密保护，但出现的场景不同。不要把它们当作登录密码，也不要把它们只存在 GranoFlow 里。

## 密钥 vs 登录密码，有什么区别

<!-- markdownlint-disable MD060 -->
|  | 登录密码（或验证邮件） | 数据密钥 / 云端同步密码 |
| --- | --- | --- |
| 用来做什么 | 证明你是谁 | 打开云端加密数据 |
| 忘了怎么办 | 重新发验证邮件 | **无法找回** |
| 换了会怎样 | 只影响登录 | 影响是否能访问云端数据 |
<!-- markdownlint-enable MD060 -->

## 数据密钥在哪里出现

创建本地备份时，GranoFlow 会在备份开始前显示“数据密钥”。它默认隐藏，点眼睛图标才显示。你需要先把它抄录并妥善保存，再勾选确认，才能开始创建备份。

这把数据密钥对应当前和之后用它创建的备份。重新生成新数据密钥，不会让旧备份改用新密钥。也就是说，旧备份仍然需要旧密钥。保存备份文件时，请同时保存当时的数据密钥。

## 云端同步密码在哪里出现

当新设备加入已有云端同步，或当前设备和云端同步数据的密钥不一致时，GranoFlow 会进入“云端同步恢复”，并要求输入“云端同步密码”。它用于尝试打开云端已有的加密同步数据。

输入正确后，GranoFlow 还会继续判断本机数据和云端数据是否属于同一套数据：

- 同一套数据：只更新这台设备的同步保护设置。
- 当前账号云端已有同步数据：以当前账号云端同步数据为准，把这台设备上可读取的本机内容带入当前账号。
- 输入不正确：不保存输入，不下载、不上传、不清空。

## 应该怎么保存

建议把密钥保存到你离开当前设备后仍能访问的地方，例如密码管理器、安全笔记、离线纸质记录或加密存储。不要只保存截图，也不要只存在 GranoFlow 里，因为需要密钥时，往往正是你换了设备、重装系统，或旧环境打不开的时候。

一个简单做法是给每条密钥写清楚用途：

- `GranoFlow 本地备份 2026-06-18 数据密钥`
- `GranoFlow 云端同步密码`
- `旧电脑迁移前备份`

这样以后导入备份或恢复同步时，你能快速判断该用哪一把。

## 在新设备上什么时候需要密钥

以下情况可能需要你输入旧设备或旧备份对应的密钥：

- 换了手机或电脑
- 重装了 GranoFlow
- 导入 `.flow.grano` 本地备份
- 看到“云端同步恢复”或云端同步密码输入提示

输入正确密钥后，新设备才有机会访问对应的备份或云端加密数据。

## 输入密钥后会发生什么

GranoFlow 会先检查这把密钥能不能打开目标数据。导入备份时，目标是备份文件；云端同步恢复时，目标是当前账号的云端同步数据。

- **密钥匹配，且数据属于同一套** → 继续导入或连接同步。
- **密钥匹配，但本机和云端不是同一套数据** → 显示选择界面，让你决定保留哪份数据。
- **密钥不对** → 不改变任何数据，让你重新输入。

## 忘了密钥怎么办

按这个顺序检查：

1. **旧设备还能用吗？** → 在旧设备上打开 GranoFlow，找到密钥并复制
2. **密码管理器里有吗？** → 检查你常用的密码管理器
3. **备份文件旁边有备注吗？** → 检查备份目录、网盘备注、安全笔记或迁移清单
4. **旧设备还在，但 App 打不开？** → 联系 GranoFlow 支持，说明旧设备和当前情况

如果以上都没有，云端加密数据可能无法恢复。本地备份（如果有的话）仍然可用。

## 看到“输入云端同步密码”时怎么办

如果 GranoFlow 提示你输入云端同步密码，请填入当前云端数据对应的完整密钥。不要先清空云端，也不要随便输入新生成的本机数据密钥来试。

输入正确后，GranoFlow 会先判断本机数据和云端数据是不是同一份：

- 如果是同一份数据，只会更新这台设备的同步密钥设置。
- 如果当前账号云端已有同步数据，会以当前账号云端数据为准，把这台设备上可读取的本机内容带入当前账号。你只需要确认当前登录的是你要继续使用的账号。

:::caution[密钥不是密码，不能重置]
数据密钥和云端同步密码丢失后，GranoFlow 无法帮你重置或找回已被它们保护的数据。现在就保存好，不要等到换设备或导入备份时才发现缺了一把钥匙。
:::
