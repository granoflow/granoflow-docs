在手机丢失、App 重装、换设备、或者误删了重要任务这些时刻，你才会真正意识到"数据安全"这件事有多重要。

GranoFlow 的数据保护体系有四层：

```text
本地存储 → 云端同步 → 端对端加密 → 手动备份
```

这四层**互相配合，但不能互相替代**。

![数据与安全设置入口](../../screenshots/zh-cn/interface-settings-account-data-entrypoints.png)

## 四层各自负责什么

| 层级 | 负责 | 不负责 |
| --- | --- | --- |
| **本地存储** | 让 App 离线也能用 | 不防止设备损坏丢失 |
| **云端同步** | 跨设备共享数据 | 不是备份，误删后云端也会删 |
| **端对端加密** | 保护云端数据不被第三方读取 | 丢了密钥自己也打不开 |
| **手动备份** | 误删、迁移时的最后防线 | 不是实时的，有时效性 |

## 最常见的误区

- **"我有云端同步，所以数据安全"**：云端同步会跟随你的操作——你删了，云端也删了。
- **"退出登录会清空数据"**：退出登录只断开账号连接，本机数据完整保留。
- **"删账号等于清除所有数据"**：账号删除的是云端数据，每台设备上的本地副本不会自动删除。

## 遇到问题先从这里判断

```text
数据还在设备上？ → 去本地恢复或导出
数据只在云端？ → 去新设备同步或下载
担心数据泄露？ → 去加密设置
担心误删？ → 去备份页面
```

:::tip[最重要的一步]
保存你的加密恢复密钥。这是唯一能让你在密钥不匹配时找回云端数据的凭证。
:::
