如果你想知道 GranoFlow 怎样保护你的数据，记住一句话：本地存储、云端同步、端对端加密、手动备份各管一件事，它们会一起帮你降低风险，但没有任何一层能替代其他层。

GranoFlow 的数据保护体系有四层：

```text
本地存储 → 云端同步 → 端对端加密 → 手动备份
```

这四层**互相配合，但不能互相替代**。手机丢失、App 重装、换设备、误删重要任务时，你需要先判断问题属于哪一层，再去对应的位置处理。

## 四层各自负责什么

| 层级 | 负责 | 不负责 |
| --- | --- | --- |
| **本地存储** | 让 App 在离线时也能使用本机数据 | 不防止设备损坏或丢失 |
| **云端同步** | 让多台设备之间共享同一份数据 | 不是备份；你误删后，云端也会跟着删除 |
| **端对端加密** | 保护云端数据不被第三方读取 | 如果密钥丢失或不匹配，你自己也打不开云端数据 |
| **手动备份** | 在误删、迁移时提供最后一道防线 | 不是实时备份；备份只包含你创建备份时的数据 |

## 最常见的误区

- **“我有云端同步，所以数据安全”**：云端同步会跟随你的操作。你删除了数据，云端通常也会同步删除。
- **“退出登录会清空数据”**：退出登录只是断开账号连接，本机数据会保留在当前设备上。
- **“删账号等于清除所有数据”**：账号删除的是云端数据。每台设备上的本地副本不会自动删除。

## 遇到问题先从这里判断

```text
数据还在设备上？ → 去本地恢复或导出
数据只在云端？ → 去新设备同步或下载
担心数据泄露？ → 去加密设置
担心误删？ → 去备份页面
```

:::tip[最重要的一步]
保存你的加密恢复密钥。这是唯一能让你在密钥不匹配时找回云端数据的凭证。
:::
