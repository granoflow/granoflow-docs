多端同步的作用是：你在一台设备上新增、修改或删除内容后，其他登录同一账号的设备也会尽量变成同样的状态。它让你可以在手机、电脑和平板之间继续同一套任务、项目、回顾和卡片工作流，但它不是用来找回误删内容的备份功能。

把同步理解成“让当前状态在多台设备之间收敛”会更准确。它关心的是现在应该一致，而不是三天前是什么样。真正需要保留历史时间点时，请先创建 [本地备份](/manual/data-security-and-recovery/backup-and-restore/)。

## 同步什么、不同步什么

✅ 会同步的内容：

- 任务，例如标题、截止日期、标签、状态等
- 项目和里程碑
- 回顾记录
- 图片和附件，在网络允许的情况下同步
- 复习卡片相关的长期资产，在当前版本支持范围内同步

⚠️ 需要特别记住：同步不是备份。

- **你删了，其他设备也会删**：同步是双向的，不会把删除操作只留在一台设备上。
- **没有版本历史**：同步不会保存「3 天前是什么样」。
- **图片可能晚一点出现**：文字内容可能先同步完成，图片和附件可能稍后才同步完成。
- **设备偏好不等于账号数据**：语言、主题、字体、部分本机设置和临时缓存通常属于当前设备，不应理解成所有设备必须一致。

## 同步的常见状态

| 状态 | 含义 |
| --- | --- |
| 同步中 | 正在上传或下载变更。 |
| 已同步 | 当前设备的数据和云端一致。 |
| 等待中 | 有变更正在排队，通常和网络有关。 |
| 错误 | 同步遇到问题，需要检查账号或密钥。 |

## 离线和服务不可达时会怎样

如果暂时没有网络，GranoFlow 里的本地数据仍然可以使用。你可以继续记录任务、整理项目、写回顾、搜索内容，也可以导出本地备份。

如果你还没有登录、不是会员，或者当前无法连接同步服务，云同步会暂时不可用。你点击需要联网的功能时，App 会提示稍后重试；本机已有数据仍然可以继续使用。

如果设备有网络，但同步服务暂时不可达，App 也不会阻止你使用本机已有的数据。已经在这台设备上的内容仍然可以编辑；需要上传到云端或从云端下载的变化，会等你稍后再次同步。

会员设备不需要单独开启静默同步。只要账号具备同步资格、服务端可达、密钥状态正常且当前设备处于可运行状态，GranoFlow 会默认尝试在后台补齐其他设备的新数据；这个过程保持静默，不会弹系统横幅，不会播放声音，也不会新增通知中心消息。

## 新设备加入同步

如果你换了新手机，或者重装了 App，想接上原来的云端数据，需要使用旧设备上的**云端同步密钥**。

详细步骤见 → [新设备同步已有云端数据](/manual/data-security-and-recovery/new-device-sync/)

## 看到“云端同步恢复”时怎么办

当 GranoFlow 发现当前设备的本机密钥和云端同步数据不匹配时，会进入“云端同步恢复”。这不是普通错误弹窗，而是在保护你避免用错误的数据覆盖云端或本机。

恢复流程会要求你输入“云端同步密码”。这个密码用来尝试打开当前账号云端数据的加密材料。输入后，GranoFlow 会先做检查：

- 如果当前账号云端已有同步数据，GranoFlow 会以当前账号的云端同步数据为准，把这台设备上可读取的本机内容带入当前账号的同步关系中。
- 如果云端数据和本机数据已经属于同一套保护关系，只会更新这台设备的同步设置，然后继续同步。
- 如果输入不正确，GranoFlow 不会保存这个输入，也不会下载云端数据、上传本机数据或清空云端。

你也可能看到“暂不同步”或“清空云端数据”一类动作。暂不同步只会关闭当前恢复入口，不会把它记成永久忽略；清空云端数据是高风险操作，会继续要求确认和系统认证。遇到不确定情况时，先暂停，确认旧设备、备份文件和当前账号的云端同步密码是否都在手边，再继续处理。

## 同步和备份应该怎么配合

日常使用时，让同步负责多设备的当前一致性；在做大操作前，让备份负责可回退副本。一个可复用的判断是：

- 今天只是想让电脑和手机看到同一批任务，用同步。
- 准备重装 App、换设备、清理大量项目或导入大量内容，先备份。
- 看到同步密钥或云端恢复提示，不要先清空云端；先确认旧设备是否还能打开、是否有本地备份、是否保存过云端同步密码。

:::caution[同步不替代备份]
请定期导出本地备份。误删的任务不能靠同步恢复，因为云端和其他设备也会跟着删除。
:::
