If you often get stuck at “which layer should this belong to,”
this page is your index.

Keep this path in mind:

> Domain → Value → Project → Milestone → Task → Review

It is not a form to fill once. It is a decision tool.

## Domain: where time ultimately goes

A domain is long-term life direction, not immediate action.

- Work and learning
- Relationships
- Health
- Creative life

Rule of thumb: if it is long-term direction, it is likely a domain.

## Value: how I want to act

A value is not a target number. It is a selection standard.

- Target: lose 5 kg in 3 months
- Value: care for my body over time, not constant overdraw

Rule: if it can be “finished,” it is usually a target, not a value.

## Project: what to push in this period

A project holds medium-term effort across days/weeks.

- Prepare an exam
- Complete a set of comics
- Build a 3-month exercise rhythm

Rule: if it needs multiple sessions and has next steps, use project.

## Milestone: which stage comes first

A milestone is project staging, not decoration.

- Complete core feature
- Fix key issues
- Prepare release assets

Rule: if you cannot tell where to continue in a project, split milestones.

## Task: what can start now

A task is the smallest action unit.

- Draft one section
- Verify login flow
- Run for 20 minutes

Rule: if you still need long thinking before first move, the task is not clear enough.

## Review: turn done into learning

Review is feedback closure, not performance theater:

- What was completed
- What worked
- What is next

Rule: if review makes next step clearer, review is effective.

## Which layer to use first

- Can do now: task first
- Multi-session effort: create project
- Project too big: split milestone
- Direction conflict: return to values
- Long-term imbalance: return to domains
