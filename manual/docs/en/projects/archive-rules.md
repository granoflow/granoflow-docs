title: "Complete and archive projects"
description: "Done with a project? Complete, archive, and delete each mean different things — pick the wrong one and you might lose tasks unexpectedly. This page explains the differences."
---

If the project is done, choose **Complete**. If you only want it out of your
current list for now, choose **Archive**. Choose **Delete** only when you are
sure the project and its contents are no longer needed.

These three actions may all feel like “putting the project away,” but they do
different things. Delete can affect the tasks or milestones inside the
project, so confirm what you actually want before using it.

## The three operations

| Operation | When to use it | What happens to tasks inside |
| --- | --- | --- |
| **Complete** | The goal has been achieved | Tasks remain and can be reviewed |
| **Archive** | Paused or hidden | Tasks remain and can be restored |
| **Delete** | Empty projects you no longer need | Hidden unless empty |

## How to access these options

Open the project detail page, then use the menu in the top-right corner to
find Complete, Archive, and Delete.

If the screenshot does not load, follow the text path instead: open the
project detail page, find the top-right more menu, then choose the action you
want.

## Deletion protection

Delete only appears for empty projects. If the project still has tasks or
milestones, the delete action is hidden; move, complete, archive, or delete
the contents first, then come back to delete the project.

This protection helps prevent you from accidentally deleting tasks or
milestones that are still useful.

:::tip[When in doubt, archive]
If you are not sure whether you will need this project later, archive is much
safer than delete. Archived projects can be restored anytime. Delete cannot be
undone.
:::

## Milestones may need to be handled before archiving

If the project still has unarchived milestones, GranoFlow may ask you to
handle those milestones before archiving the entire project. This prevents the
project from being put away while it still contains phases that appear to be
in progress.

If you see a message like “Please archive milestones first”, go back to the
project detail, complete or archive each milestone; once all phases are
closed, archive the project.

## Archive vs complete — which to choose

- The goal was **achieved**: choose “Complete”. This records the project’s
  completion date
- The goal is **on hold**, or you just do not want to see it every day:
  choose “Archive”. It does not mean success; it only puts the project away
- The project is **no longer needed at all**, and you do not need the contents
  either: only then choose “Delete”
