title: "Create a project"
description: "Create a project when you want to manage a group of tasks together. This page explains where to create one, what to fill in, and what you can do next."
---

If you want to manage a larger goal, such as a launch, a course, or a plan for
a period of time, create a project: open the projects page, tap **+** in the
top-right corner, enter a project name and a brief description, then create
it.

## Where to start

In the project list, tap the **+** button in the top-right corner. A creation
dialog appears.

![Projects list entry point](../../../screenshots/en/projects-overview-main.png)

You can fill in:

- **Name** (required): describe what this project is meant to complete.
  For example, “Q3 product launch” is easier to understand than “Project 1”.
- **Description** (optional): record background, scope, constraints, or
  delivery notes. It stays as project context when you open the project later.
- **Domain** (optional): place the project under a broader area, such as Work,
  Learning, or Health.

:::tip[A good name saves time later]
The project name appears in your project list. A “verb + goal” format works
well, such as “Finish thesis” instead of just “Thesis”.
:::

## Create an empty project vs add tasks first

Both approaches work. Choose the one that fits how you work:

| Approach | Best when |
| --- | --- |
| Create the project first | You are still planning the structure |
| Capture in Inbox first | You want to record quickly, then organize later |

Either way, tasks can end up managed inside the project.

## After creating the project

After the project is created, you can usually continue with these actions:

1. Add milestones (optional) — if the goal has clear phases.
2. Link existing tasks to a project and a milestone.
3. Create new tasks directly under a milestone.

If you are not sure what tasks to add yet, you can close the project page for
now. The project has already been created, and you can come back later to fill
it in.
