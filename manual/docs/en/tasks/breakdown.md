Break complex tasks into executable steps, and understand how node completion, parent task status, and ordering limits relate.

## Where To Start

Open task details and add child nodes in the breakdown or steps area. This works best for tasks that need several actions before they are done.

## How To Use It

- Write the next action as a short sentence, then add more nodes when needed.
- Completing child nodes changes the parent task's progress. When all effective child nodes are complete, the parent task can be treated as complete.
- If you add or restore an unfinished node, the parent task may return to an active state so complex work is not closed too early.

## Results And Boundaries

Breakdown does not create another project. It makes one task more executable. Ordering and dragging may be limited by state so completed or protected content is not reordered casually.

- Without child nodes, the task behaves like a normal task.
- Completing nodes is not the same as completing a project; projects and milestones have their own states.

## Next Step

When you need stage goals, use "Manage milestones" instead of endlessly nesting subtasks.
