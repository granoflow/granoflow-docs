This chapter turns ACT and The Happiness Trap into a GranoFlow practice: domains, values, projects, milestones, tasks, and review. Use it to turn long-term goals into one task you can start today, without making task management another source of anxiety.

GranoFlow is not only a tool for listing todos.

It is closer to a personal time structure: first notice the directions you care about over time, then turn those directions into projects, milestones, and tasks, and finally use review to turn experience into accumulation.

If you came here from *The Happiness Trap* or ACT (Acceptance and Commitment Therapy), this chapter shows how to place "noticing values" and "committed action" inside GranoFlow's structure. For the full background, read [ACT and The Happiness Trap](/en/value-to-action/act-loop/).

You do not need to set up all of these concepts at the beginning. Understand how they relate first, then fill in the structure gradually.

## The Structure in One Line

You can first understand GranoFlow as a path that moves from large to small, then from small back to large:

> domains -> values -> projects -> milestones -> tasks -> review

Domains show which life direction your time goes into.  
Values show how you want to act.  
Projects hold goals that continue over a period of time.  
Milestones break projects into stages.  
Tasks are steps you can take today.  
Review connects completed actions back to your long-term direction.

This is not a strict form you must fill out. It is a language for organizing life.

## Domains: Long-Term Life Directions

A domain is a life direction where you invest time and attention over the long term.

Common domains include:

- work and learning
- relationships
- physical and mental health
- creative work

A domain is not a short-term goal, and it is not a task folder. It is more like a long-running direction.

For example, "finish one app version" is not a domain. It fits better as a project.  
"Work and learning" is a domain.  
"Run three times a week" is not a domain. It fits better as a task or habit plan.  
"Physical and mental health" is a domain.

Domains help you see where your time ultimately goes.

If you spend several weeks heavily on work and learning while barely caring for your body, review can show that shift. GranoFlow will not decide whether that is right or wrong for you, but it helps you see reality clearly.

## Values: How You Want to Act

Values are not goals.

Goals can be completed. Values cannot be completed once and for all.

For example:

> Lose 5 kg in three months

That is a goal.

> I want to care for my body over time instead of always overusing it

That is a value.

Another example:

> Release a product version

That is a goal.

> I want to be reliable and keep delivering

That is a value.

Values provide direction when you make choices. They do not tell you exactly which task you must do today, but they remind you which actions are closer to the person you want to become.

Your values do not need to sound grand. The more ordinary and real they are, the easier they are to keep using.

## Projects: Work That Continues for a While

A project is larger than a task and more concrete than a life goal.

It usually lasts days, weeks, or even months.

For example:

- prepare for an exam
- finish a product version
- build a personal website
- complete a set of comics
- build an exercise plan for a period of time

A project should answer one question:

> What do I want to keep moving forward during this period?

If something can be finished today, it usually does not need to become a project. Write it as a task.

If something repeatedly takes your attention and needs to be split, moved forward, paused, and resumed, it is a good project candidate.

## Milestones: Stage Markers inside a Project

A milestone is a stage goal inside a project.

Its job is not to add complexity. It makes a larger project easier to move forward.

For example, a project called "finish a product version" could be split into:

- finish core features
- fix major issues
- prepare release materials
- submit for review

Each milestone answers one question:

> Which part should this stage finish first?

Without milestones, a project can become a vague large goal.  
With milestones, you know that you are not trying to "finish the whole project" right now. You are moving the current stage forward first.

Small projects can have no milestones. Larger projects usually benefit from them.

## Tasks: One Step You Can Take Today

Tasks are the basic action unit in GranoFlow.

A good task should be something you can start directly.

For example:

- finish the first draft of homepage copy
- check the sign-in flow
- organize 10 pieces of test feedback
- send a confirmation email to someone
- run for 20 minutes

Weak tasks are often too large, too vague, or too much like wishes.

For example:

- become more disciplined
- make the product good
- learn English well
- improve life

These usually need to become projects or values first, then be broken into tasks.

The point of a task is not to sound polished. It is to show you what the next step is.

If a task keeps stopping you from starting, it usually does not mean you are lazy. It usually means the task is not concrete enough. Keep breaking it down until it becomes an action you can take.

## Review: Turn Experience into Accumulation

Review is a very important part of GranoFlow.

After a task is completed, without review, it is only an item crossed off a list.  
With review, it can become experience, judgment, and accumulation.

Review can be short.

You only need to ask:

- What did I complete today?
- Which actions moved me closer to the direction I care about?
- Where did I drift away?
- What is the next step?

Review is not for criticizing yourself or creating pressure.

It helps you see what actually happened. What worked can be kept. What did not work can be adjusted. Projects that no longer matter can be let go.

## How They Connect

Here is a complete example.

You have a domain:

> creative work

Your value in this domain is:

> I do not want to only consume content. I want to keep expressing and creating.

So you create a project:

> Finish the first set of GranoFlow comics

The project can be split into milestones:

> Decide the theme  
> Write the script  
> Generate images  
> Publish the first version

Today's task might be:

> Write copy for the 4 panels in the first comic set

During evening review, you can write:

> I finished the first script draft today, but the cover hook is not strong enough. Tomorrow I will revise the first image first.

Now a small task is connected to a long-term domain and value.

It is no longer only "I did a todo today." It is a small step toward the kind of person you are trying to become.

## You Do Not Need the Full Structure at First

When you start using GranoFlow, you do not need to create all domains, values, projects, and milestones immediately.

The simplest path is:

1. Write tasks first.
2. When a task keeps continuing, create a project.
3. When a project gets larger, split it into milestones.
4. During review, gradually organize which domain and value it belongs to.

The structure grows slowly. It is not something you finish on the first day.

You can start from one task.  
You can start from one project.  
You can start from one review.

If it helps you see the next step more clearly, it is already working.
