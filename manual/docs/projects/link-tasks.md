把任务连接到项目后，它会出现在项目里；如果它有日期，也仍然会出现在今日视图或日历视图里。项目用来说明这件事属于哪个目标，里程碑用来说明它属于项目里的哪个阶段。

这里最容易误会的是收集箱：**连接项目本身不会让一个无日期任务自动离开收集箱**。收集箱主要看任务有没有日期；如果任务仍然没有日期，它可能会继续留在收集箱里，方便你稍后安排时间。

## 怎么连接

你可以用几种方式把任务连接到项目。

### 方法一：在任务详情里选择项目

打开任务详情，找到项目字段，选择目标项目。需要时，也可以继续选择这个项目下的某个里程碑。

这是最常用的方式，适合你已经打开任务、想直接给它归类的时候。

### 方法二：在任务列表或收集箱里“加入项目”

在任务列表或收集箱里使用“加入项目”时，需要选择一个具体里程碑。这样任务会同时连接到项目和这个阶段。

### 方法三：在项目页面创建新任务

打开项目详情页，在某个里程碑下新增任务。这样创建出来的任务会自动带上当前项目和里程碑，不需要再手动选择一次。

## 任务连接后出现在哪里

连接项目之后，同一个任务可能会同时出现在多个地方。它不是被复制了，而是同一个任务在不同视图里显示。

| 视图 | 说明 |
| --- | --- |
| 项目页面 | 出现在对应项目里；如果选了里程碑，会出现在对应里程碑下 |
| 今日视图 | 如果任务有今天的日期，仍然出现 |
| 日历视图 | 按截止日期显示 |
| 收集箱 | 如果任务仍然没有日期，并且还是待办或进行中，可能继续出现 |

:::note[收集箱的变化]
收集箱不是“没有项目的任务列表”，而是“还没有安排日期的待处理任务列表”。如果你把任务加入了带截止日期的里程碑，GranoFlow 可能会把里程碑日期带到任务上；这时它会按日期进入任务列表或日历，不再留在收集箱。
:::

## 挂到项目和挂到里程碑有什么区别

挂到项目，表示这个任务属于这个项目。

挂到里程碑，表示这个任务不只属于项目，还属于项目里的某个阶段。比如一个项目里有“准备”“执行”“复盘”几个里程碑，任务可以放到其中一个里程碑下。

在任务列表或收集箱里的“加入项目”操作通常会要求你选到具体里程碑。这样做是为了避免任务只挂在一个大项目下面，却不知道属于哪个阶段。

## 想把一个任务从项目里移出来

打开任务详情，把项目字段清空就行。

清空后，如果这个任务也没有截止日期，并且还是待办或进行中，它会重新出现在收集箱。如果它还有截止日期，它仍然会按日期出现在今日视图或日历视图里。

## 一个任务能挂多个项目吗

不能。每个任务只能属于一个项目（和一个里程碑）。

如果你发现一个任务好像跨了两个项目，通常有两种处理方式：

1. 选择它更主要归属的那个项目
2. 把它拆成两个任务，分别挂到各自的项目
