---
title: "管理里程碑"
description: "里程碑是项目里的阶段检查点，帮你把一个大目标拆成可以判断进展的几段。"
---

项目变大以后，最容易让人焦虑的不是任务很多，而是你不知道自己到底推进到哪一步了。任务告诉你“接下来做什么”，里程碑告诉你“这一段做到什么程度才算过关”。

所以，在 GranoFlow 里，里程碑不只是把任务分组的文件夹。它更像项目里的阶段检查点：一个里程碑下面可以放多项任务，但这个阶段真正要回答的是“这个结果有没有达成”。当你这样理解里程碑时，项目页面就不再是一条很长的待办清单，而是一张能看出阶段进度的路线图。

假设你在准备一次课程发布。你可以把项目拆成“课程大纲确定”“录制完成”“上线检查”几个里程碑。每个里程碑下面放对应任务，例如写大纲、录制单元、检查链接、准备发布文案。这样你每次打开项目时，看到的不是“还有 27 个任务”，而是“现在卡在录制完成之前，下一步要把这一段收掉”。

## 先想清楚阶段结果

创建里程碑前，先问一个简单问题：这个阶段完成以后，别人能看见什么结果？

好的里程碑名称通常像一个已经交付的结果，比如“初稿完成”“测试通过”“课程上线”。它不一定要很长，但最好能让你过几天回来时仍然看懂它的意义。

尽量少用“第一阶段”“第二阶段”这种名称。它们在刚创建时看起来很整齐，但过一段时间以后，你还要重新回忆第一阶段到底指什么。里程碑本来就是为了减少这种回忆成本。

## 添加里程碑

进入项目详情页后，点击“添加里程碑”，输入名称，再点击“添加”。新里程碑创建后，会出现在项目里，你就可以把相关任务放到它下面。

新增里程碑这一步需要点“添加”，因为它是在创建一个新的阶段节点。创建完成以后，再编辑这个已有里程碑时，保存方式会变得不同：多数修改会在你完成对应动作后自动生效，不需要再点一个额外的保存按钮。

## 编辑已有里程碑时，什么时候会保存

已有里程碑的编辑采用自动保存。你改完某个字段、完成对应动作后，GranoFlow 会把修改写入项目，不需要在页面底部再找“储存”或“保存”按钮。

不同内容的生效时机略有区别：

| 你修改的内容 | 什么时候生效 |
| --- | --- |
| 标题 | 修改后离开标题输入框时保存 |
| 截止日期 | 选择日期后保存 |
| 描述 | 在描述编辑页保存后生效 |
| 图片 | 上传或删除图片后生效 |
| PDF 附件 | 上传或删除 PDF 后生效 |

这里容易误解的一点是：自动保存并不表示你还没完成输入时就一定已经提交。比如标题通常是在你离开输入框时保存；描述会在描述编辑页里保存后回到里程碑。你只需要按正常编辑节奏完成这个动作，不需要额外再点一次总保存。

如果你只是新建一个里程碑，仍然要点击“添加”来完成创建。自动保存说的是“已有里程碑”的后续修改。

## 右侧编辑页怎么关闭

在宽屏或桌面布局里，点击里程碑后，右侧会打开里程碑编辑页。它不是一个必须提交的弹窗，而是项目详情的一块侧边编辑区域。

想收起右侧编辑页时，点击左侧项目详情里的空白区域即可。这样不会改变你的项目内容，只是把右侧编辑页收起来，让你回到项目整体视图。

如果你点击的不是空白区域，而是左侧的其他项目、里程碑、任务或操作按钮，GranoFlow 会执行你点的动作。比如你点另一个里程碑，右侧会切换到那个里程碑；你点导航栏切到其他页面，当前里程碑编辑页会一起关闭。

在窄屏或手机布局里，里程碑编辑页会像一个新页面一样打开。此时使用顶部返回按钮回到项目详情页。

## 用状态判断这个阶段是否收尾

里程碑状态来自它下面的任务进度。你不需要手动把一个阶段改成“已完成”；当里程碑里的任务都完成后，它就会进入完成状态。

| 状态 | 含义 |
| --- | --- |
| 进行中 | 里程碑里还有未完成的任务 |
| 已完成 | 里程碑里的所有任务都完成了 |
| 已归档 | 你手动归档后，它会从当前视图收起 |

这个规则有一个重要边界：如果一个已经完成的里程碑后来又加入了未完成任务，它会回到“进行中”。这不是状态出错，而是 GranoFlow 在提醒你：这个阶段又有新的未完成内容了。

## 调整里程碑顺序

里程碑会按照项目详情页里的排列顺序显示。你可以拖拽里程碑来调整顺序。

通常建议按项目推进顺序排列：先发生的阶段放在上面，最后交付的阶段放在下面。这样从上往下看，就是项目从准备到交付的路线。只有当你的项目更适合按主题组织时，才把同类阶段放在一起。

## 删除前先确认它是空的

只有空里程碑会显示删除入口。如果里面还有任务，删除按钮不会出现。你需要先移动、完成、归档或删除这些任务，再回来删除这个里程碑。

:::caution[删除里程碑不会自动删除任务]
删除里程碑不会连带删除任务。先处理任务，再删除空里程碑。
:::

理解了里程碑以后，下一步通常是把任务放进合适的项目和阶段。继续阅读“把任务连接到项目”，你会看到任务挂到项目、挂到里程碑，以及它是否还会出现在今日视图里的区别。
