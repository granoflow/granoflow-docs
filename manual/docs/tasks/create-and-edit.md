添加任务的规则只有一条：**写个标题，保存**。其他字段都是可选的，等你需要的时候再填。

## 从哪里创建任务

| 入口 | 适合的场景 |
| --- | --- |
| 底部 **+** 按钮 | 随时随地的快速录入 |
| 收集箱页面内的输入框 | 专门整理收集箱时 |
| 项目或里程碑页面内 | 直接归属到某个项目 |
| 已有任务详情里的节点 | 拆分一个大任务的步骤 |

## 任务编辑界面

![创建与编辑任务对话框](../../screenshots/zh-cn/tasks-create-edit-dialog.png)

各字段说明：

| 字段 | 是否必填 | 作用 |
| --- | --- | --- |
| 标题 | ✅ 必填 | 任务的名称，越具体越容易执行 |
| 描述 | 可选 | 背景信息、链接、备注 |
| 截止日期 | 可选 | 设置后任务出现在对应日期的任务列表 |
| 提醒 | 可选 | 在指定时间发通知，不能设在过去 |
| 项目 | 可选 | 设置后任务从收集箱移入该项目 |
| 里程碑 | 可选 | 设置后归属到项目的某个阶段 |
| 标签 | 可选 | 方便筛选，可以有多个 |
| 节点 | 可选 | 把任务拆成更小的步骤 |

:::tip[善用自然语言输入]
在标题输入框里，你可以直接写 `#标签名`、`@日期`、`~提醒时间`，GranoFlow 会自动解析。比如输入 `整理报告 @明天 #工作` 会自动识别出明天的日期和"工作"标签。详细的解析规则见[用自然语言写任务](title-parser)。
:::

## 保存后任务去哪了

这取决于你填了什么：

- **没有日期、没有项目** → 进收集箱
- **有日期** → 出现在那一天的任务列表
- **有项目** → 出现在项目里
- **在项目页面里创建** → 直接归属到那个项目

修改字段不是新建一个任务，而是改变同一个任务的归属。

## 编辑已有任务

点击任何任务即可打开详情页。修改你想改的字段，退出时自动保存。

:::caution[注意]
提醒不能设置在已经过去的时间。如果你设的提醒时间已经过了，系统会提示你重新选。
:::

完成、归档和删除是三种不同操作，填写字段不会让任务自动变成完成状态。
