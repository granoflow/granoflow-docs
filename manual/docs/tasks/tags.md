---

想给任务按「状态、类型、精力、临时重点」这类角度做标记，就用标签。项目说明任务属于哪个目标；标签说明这个任务有什么共同特征，方便你以后筛选和整理。

比如你有「健身 App 开发」这个项目，里面的任务有些是「需要设计稿」，有些是「等待第三方 API」。这些不是新的项目，而是跨项目都可能出现的状态或类型，就适合用标签。

## 怎么给任务加标签

在新建任务或打开任务详情时，找到标签区域，然后选择已有标签，或输入新名称创建标签。

![标签管理界面](../../screenshots/zh-cn/tasks-tags-management.png)

即使截图没有加载，也可以按下面理解：

- 已有标签会显示为可选项
- 找不到合适的标签时，可以输入一个新名称来创建
- 一个任务可以同时加多个标签

## 自定义标签模板

在标签管理里编辑已有自定义标签时，可以给它补充描述模板和节点模板。之后任务选择这个标签时，GranoFlow 会把模板内容复制到这条任务自己的描述和节点里。

模板是一次性复制：复制后内容就属于任务本身。以后修改或删除标签模板，不会自动改写已经创建过的任务。

内置示例 `account`、`bug` 和 `feature` 都是普通自定义标签。`account` 适合账号整理类任务，默认描述模板包含平台名称、网址、邮箱、手机、第三方联合登录和备注，不包含密码字段；`bug` 适合记录复现步骤、预期结果和实际结果；`feature` 适合记录用户价值、使用场景和验收标准。

## 标签用来记什么好

标签适合表达**跨项目、会重复使用**的分类。下面这些都比较适合：

| 用途 | 标签示例 |
| --- | --- |
| 场景 / 精力 | `低精力` `深度工作` `碎片时间` |
| 等待状态 | `等待他人` `等待回复` `待确认` |
| 类型 | `电话` `创意` `管理` |
| 临时标记 | `本周重点` `稍后处理` |

不建议把项目名称复制成标签。任务已经在项目里时，不需要再用同一个项目名标一遍。

## 删除标签的影响

删除一个标签**不会**删除使用它的任务。它只会把这个标签从那些任务上移除。

:::caution[删前确认]
删除标签是不可撤销的。确认不再需要这个分类，再操作。
:::

## 标签多了怎么办

标签太多会让筛选变得没用。建议定期整理：

- 合并意思接近的标签
- 删除已经没人用的标签
- 保持标签名称简短、容易区分
