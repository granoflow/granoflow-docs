不想每次都在字段里一个个点选？试试在标题里直接写：

- `#工作` — 添加一个叫"工作"的标签
- `@明天` — 设置截止日期为明天
- `~下午3点` — 设置提醒时间

GranoFlow 会把这些识别出来，显示成候选项，等你确认。

## 怎么操作

输入标题时，只要写了 `#` `@` `~` 或日期词，输入框会高亮显示识别结果。

![自然语言解析确认界面](../../screenshots/zh-cn/tasks-title-parser-confirmation.png)

确认候选项的方式：

- 点击高亮区域的候选项
- 按 **Enter** 或 **Tab** 确认当前候选

:::caution[重要：确认前不会写入]
识别到的内容**不会**自动变成字段。你必须明确确认，才会写入标签、日期或提醒。没有确认的部分会继续留在标题文字里，不会偷偷消失。
:::

## 常用语法速查

| 你输入的 | 效果 |
| --- | --- |
| `#工作` | 候选添加"工作"标签 |
| `@明天` `@下周五` `@3月5日` | 候选设置截止日期 |
| `~下午3点` `~明早9点` | 候选设置提醒时间 |
| `写报告 @明天 #工作` | 同时候选日期和标签 |

## 有什么限制

- `#` `@` `~` 需要在清楚的词边界前使用，比如 `#工作` 而不是 `搞#工作`
- 解析结果只是建议，最终以你确认的字段为准
- 复杂的句子里夹杂的符号不一定能被正确识别

如果哪次识别结果不对，直接在字段里手动设置就好，不用一定依赖解析。
