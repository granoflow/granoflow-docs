GranoFlow 里记录的是任务、项目、回顾、价值观——很私人的东西。这页帮你弄清楚几个最重要的概念，不讲复杂技术，只讲你真正需要知道的事。

## 本地优先是什么意思

本地优先 = 数据先存在你的设备上，不需要联网才能用。

记录任务、整理项目、查看回顾——这些都在本机完成，没有网络也没关系。如果你开启云同步，GranoFlow 才会把数据上传到云端，用来多设备同步和换设备后恢复。

> 本地 = 你正在用的工作区  
> 云端 = 多设备同步和恢复用的加密副本

## 端到端加密保护什么

端到端加密（E2EE）的意思是：数据在离开你的设备之前，就已经加密了。云端存的是密文，GranoFlow 的服务器读不到你的任务内容。

但加密不是魔法。当你在设备上正常使用时，App 需要在本地解密才能显示内容。如果设备本身被恶意软件控制，或者有人拿到了你的设备解锁权限，端到端加密也挡不住。

所以你还是要：锁好屏幕、妥善保存密钥、不随便把内容复制给不可信的服务。

## 密钥不是登录密码（最常见的误解）

这是最容易搞错的地方：

| | 用途 |
|--|------|
| **登录密码 / 邮箱验证码** | 证明账号属于你 |
| **加密密钥** | 解开你的加密数据 |

你可以顺利登录账号，但如果缺少正确密钥，云端或备份里的数据仍然解不开。

这不是 GranoFlow 故意为难你——这是端到端加密的基本代价：如果服务器能随便帮你解开数据，那就说明它本来就能读你的内容。

**密钥保存建议：**
- 存进可靠的密码管理器
- 不要只截图存在一台设备里
- 换设备前确认密钥可用

## 忘记密钥会怎样

- 有旧设备能打开数据 → 可以继续用或重新建立同步
- 只有云端加密数据，没有密钥 → 服务器无法替你解开
- 有备份文件，没有创建备份时的密钥 → 备份也无法恢复

> 登录让你回到账号，密钥让你解开数据。这两件事不能互相替代。

## 换设备前要做什么

1. 旧设备仍然能打开 GranoFlow
2. 云同步状态正常，或导出了最新备份
3. 确认知道当前数据对应的密钥

然后：新设备登录同一账号 → 按提示输入密钥 → 恢复或同步。

:::caution[空设备不是数据来源]
新设备第一次同步时，不要把一台没有数据的设备当成"新起点"同步到云端——这会覆盖掉你的旧数据。
:::
