備份係你的「後悔藥」。但佢唔會自動生效——你需要主動創建，主動保存到安全的地方。

![備份與恢復管理界面](../../../screenshots/zh-hk/data-backup-restore-management.png)

## 備份 vs 同步，有什麼區別

| | 備份 | 雲端同步 |
|--|------|---------|
| 記錄歷史狀態？ | ✅ 係某個時間點的快照 | ❌ 只有當前狀態 |
| 誤刪後能恢復？ | ✅ 可以恢復到備份時間點 | ❌ 刪除會同步到雲端 |
| 需要主動操作？ | ✅ 需要手動導出 | ✅/❌ 自動進行，但唔保存歷史 |

## 什麼時候應該做備份

- 升級 App 大版本之前
- 換手機或重裝系統之前
- 刪除大量任務或項目之前
- 完成一個重要階段後想留存記錄

## 點做備份

1. 進入 GranoFlow 設定 → 數據/備份
2. 選擇導出備份
3. 把備份文件保存到你能控制的位置（iCloud、本地文件夾、電腦……）

## 點從備份恢復

1. 進入 GranoFlow 設定 → 數據/備份
2. 選擇導入備份
3. 搵到備份文件並導入

:::caution[恢復會覆蓋當前數據]
從備份恢復係一個覆蓋操作——導入後，當前設備的數據會被備份文件替換。如果你要保留當前設備的最新內容，請先導出當前備份，再導入舊備份。
:::
