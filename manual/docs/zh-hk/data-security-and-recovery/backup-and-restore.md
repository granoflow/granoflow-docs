如果你想在誤刪、換設備或重裝前保留一份可回退的數據，請在 GranoFlow 設定裡的數據/備份頁面手動匯出備份檔案，並把檔案儲存到你自己能找到、能控制的位置。

## 備份和同步有什麼區別

備份是一份「某個時間點的數據副本」。同步是把當前數據同步到雲端或其他設備。它們解決的問題不一樣。

<!-- markdownlint-disable MD060 -->
|  | 備份 | 雲端同步 |
| --- | --- | --- |
| 是否保留歷史狀態？ | ✅ 是某個時間點的快照 | ❌ 只代表當前狀態 |
| 誤刪後能否回到舊狀態？ | ✅ 可以恢復到備份建立時的狀態 | ❌ 刪除通常也會同步到雲端 |
| 是否需要你主動操作？ | ✅ 需要手動匯出並儲存檔案 | ✅/❌ 同步會自動進行，但不儲存歷史版本 |
<!-- markdownlint-enable MD060 -->

## 什麼時候應該做備份

建議在這些時候先匯出一份備份：

- 升級 App 大版本之前
- 換手機、換電腦或重裝系統之前
- 刪除大量任務或項目之前
- 完成一個重要階段後，想保留當時的記錄

## 卡片盒包與完整備份

數據管理頁以平鋪卡片展示：`本地備份` 卡片匯出 `.flow.grano` 時會用設備密鑰加密，適合換機或整機恢復；`卡片盒` 卡片處理 `.deck.grano` 卡片盒包，可遷移選定卡片盒、卡片和可打包的本地圖片媒體，但不能替代完整備份，也不會自動同步到雲端。卡片盒卡片還會顯示當前卡片快取佔用與上限，並提供清空快取入口。更多邊界閱讀 [卡片盒、匯入與匯出](/manual/zh-hk/review-cards/decks-import-export/)。

在「卡片盒」卡片點「匯出當前卡片盒」會進入卡片盒列表，由你在列表裡選定頂層卡片盒後再匯出。

## 怎麼做備份

1. 開啟 GranoFlow 設定。
2. 進入數據管理頁面。
3. 在「本地備份」卡片選擇「建立本地備份包」。
4. 等待匯出完成，不要在處理中重複點擊或關閉頁面。
5. 把匯出的備份檔案儲存到你能控制的位置，例如 iCloud、本地資料夾或電腦。

## 怎麼從備份恢復

1. 開啟 GranoFlow 設定。
2. 進入數據/備份相關頁面。
3. 選擇匯入備份。
4. 找到之前儲存的備份檔案。
5. 確認匯入後，等待恢復完成，不要在處理中重複操作。

:::caution[恢復會覆蓋當前數據]
從備份恢復是覆蓋操作。匯入後，當前裝置上的數據會被備份檔案裡的數據取代。如果你想保留當前裝置的最新內容，請先匯出一份當前備份，再匯入舊備份。
:::
