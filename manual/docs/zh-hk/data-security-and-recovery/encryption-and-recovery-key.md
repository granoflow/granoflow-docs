你現在最需要做的是：在還能打開 GranoFlow 的設備上，把關鍵密鑰保存到密碼管理器或安全的地方。以後換設備、重裝 App、導入本地備份，或看到「雲端同步恢復」時，GranoFlow 可能會要求你輸入對應密鑰；沒有它，某些加密數據可能打不開。

![加密與恢復密鑰 界面截圖](../../screenshots/zh-hk/data-encryption-recovery-key.png)

GranoFlow 的備份和雲端同步都使用加密。密鑰像保險箱鑰匙：**沒有正確的鑰匙，連 GranoFlow 自己的伺服器也無法讀取被保護的數據**。這也意味着：**如果你自己把密鑰弄丢了，GranoFlow 無法幫你重置或找回已經被它保護的數據。**

這裏有兩個用戶會看到的名字：創建本地備份時顯示的是「數據密鑰」；雲端同步恢復裏輸入的是「雲端同步密碼」。它們底層都服務於加密保護，但出現的場景不同。不要把它們當作登錄密碼，也不要把它們只存在 GranoFlow 裏。

## 密鑰 vs 登錄密碼，有什麼區別

<!-- markdownlint-disable MD060 -->
|  | 登錄密碼（或驗證郵件） | 數據密鑰 / 雲端同步密碼 |
| --- | --- | --- |
| 用來做什麼 | 證明你是誰 | 打開雲端加密數據 |
| 忘了怎麼辦 | 重新發驗證郵件 | **無法找回** |
| 換了會怎樣 | 只影響登錄 | 影響是否能訪問雲端數據 |
<!-- markdownlint-enable MD060 -->

## 數據密鑰在哪裏出現

創建本地備份時，GranoFlow 會在備份開始前顯示「數據密鑰」。它預設隱藏，點眼睛圖標才顯示。你需要先把它抄錄並妥善保存，再勾選確認，才能開始創建備份。

這把數據密鑰對應當前和之後用它創建的備份。重新生成新數據密鑰，不會讓舊備份改用新密鑰。也就是說，舊備份仍然需要舊密鑰。保存備份文件時，請同時保存當時的數據密鑰。

## 雲端同步密碼在哪裏出現

當新設備加入已有雲端同步，或當前設備和雲端同步數據的密鑰不一致時，GranoFlow 會進入「雲端同步恢復」，並要求輸入「雲端同步密碼」。它用於嘗試打開雲端已有的加密同步數據。

輸入正確後，GranoFlow 還會繼續判斷本機數據和雲端數據是否屬於同一套數據：

- 同一套數據：只更新這台設備的同步保護設定。
- 不是同一套數據：以當前帳號的雲端保護為目標，把本機數據遷入這個帳號。
- 輸入不正確：不保存輸入，不下載、不上傳、不清空。

## 應該怎麼保存

建議把密鑰保存到你離開當前設備後仍能訪問的地方，例如密碼管理器、安全筆記、離線紙本記錄或加密存儲。不要只保存截圖，也不要只存在 GranoFlow 裏，因為需要密鑰時，往往正是你換了設備、重裝系統，或舊環境打不開的時候。

一個簡單做法是給每條密鑰寫清楚用途：

- `GranoFlow 本地備份 2026-06-18 數據密鑰`
- `GranoFlow 雲端同步密碼`
- `舊電腦遷移前備份`

這樣以後導入備份或恢復同步時，你能快速判斷該用哪一把。

## 在新設備上什麼時候需要密鑰

以下情況可能需要你輸入舊設備或舊備份對應的密鑰：

- 換了手機或電腦
- 重裝了 GranoFlow
- 導入 `.flow.grano` 本地備份
- 看到「雲端同步恢復」或雲端同步密碼輸入提示

輸入正確密鑰後，新設備才有機會訪問對應的備份或雲端加密數據。

## 輸入密鑰後會發生什麼

GranoFlow 會先檢查這把密鑰能不能打開目標數據。導入備份時，目標是備份文件；雲端同步恢復時，目標是當前賬號的雲端同步數據。

- **密鑰匹配，且數據屬於同一套** → 繼續導入或連接同步。
- **密鑰匹配，但本機和雲端不是同一套數據** → 使用當前帳號的雲端保護作為目標，GranoFlow 會把本機數據遷入這個帳號。
- **密鑰不對** → 不改變任何數據，讓你重新輸入。

## 忘了密鑰怎麼辦

按這個順序檢查：

1. **舊設備還能用嗎？** → 在舊設備上打開 GranoFlow，找到密鑰並複製
2. **密碼管理器裏有嗎？** → 檢查你常用的密碼管理器
3. **備份文件旁邊有備註嗎？** → 檢查備份目錄、網盤備註、安全筆記或遷移清單
4. **舊設備還在，但 App 打不開？** → 聯絡 GranoFlow 支援，說明舊設備和當前情況

如果以上都沒有，雲端加密數據可能無法恢復。本地備份（如果有的話）仍然可用。

## 看到「輸入雲端同步密碼」時怎麼辦

如果 GranoFlow 提示你輸入雲端同步密碼，請填入當前雲端數據對應的完整密鑰。不要先清空雲端，也不要隨便輸入新生成的本機數據密鑰來試。

輸入正確後，GranoFlow 會先判斷本機數據和雲端數據是不是同一份：

- 如果是同一份數據，只會更新這台設備的同步密鑰設定。
- 如果不是同一份數據，會以當前帳號的雲端保護為目標。繼續前請先確認你正在使用預期的帳號。

:::caution[密鑰不是密碼，不能重置]
數據密鑰和雲端同步密碼丟失後，GranoFlow 無法幫你重置或找回已被它們保護的數據。現在就保存好，不要等到換設備或導入備份時才發現缺了一把鑰匙。
:::
