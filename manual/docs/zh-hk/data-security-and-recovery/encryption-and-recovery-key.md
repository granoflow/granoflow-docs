你現在最需要做的是：在還能打開 GranoFlow 的設備上，把關鍵密鑰保存到密碼管理器或安全的地方。以後換設備、重裝 App、恢復本地備份，或看到「雲端同步恢復」時，GranoFlow 可能會要求你輸入對應密鑰；沒有它，某些加密數據可能打不開。

![加密與恢復密鑰 界面截圖](../../screenshots/zh-hk/data-encryption-recovery-key.png)

GranoFlow 的備份和雲端同步都使用加密。密鑰就像保險箱鑰匙：**沒有正確的鑰匙，連 GranoFlow 自己的伺服器也無法讀取受保護的數據**。這也代表：**如果你自己遺失密鑰，GranoFlow 無法幫你重置或找回已經被它保護的數據。**

你會看到兩個用戶介面名稱：建立本地備份或首次上傳雲端同步前，畫面顯示的是「數據密鑰」；雲端同步恢復裏輸入的是「雲端同步密碼」。它們底層都服務於同一種加密保護，但名稱會跟着使用目標變化：打開備份時叫數據密鑰，打開當前帳號雲端同步數據時叫雲端同步密碼。

如果你從未更改過密鑰，同一串 `GF1-...` 可能既是某次備份的數據密鑰，也是當前雲端同步密碼。為了日後不混淆，保存時仍建議寫清楚它來自哪一次備份、哪一次首次同步，或哪一次重置。

## 先分清三個名稱

<!-- markdownlint-disable MD060 -->
|  | 登入密碼（或驗證電郵） | 數據密鑰 | 雲端同步密碼 |
| --- | --- | --- | --- |
| 用來做甚麼 | 證明你是誰 | 打開本地備份，或在首次同步時成為雲端保護密鑰 | 打開當前帳號已有的雲端同步數據 |
| 甚麼時候看到 | 登入帳號時 | 建立本地備份、首次把本機數據同步到雲端，或修改數據密鑰時 | 新設備加入已有雲端同步，或本機密鑰和雲端不匹配時 |
| 忘記了怎麼辦 | 可以重新發送驗證電郵 | **無法找回** | **無法找回** |
| 更改後會怎樣 | 只影響登入 | 之後建立的備份和之後的雲端保護設定會用新密鑰；舊備份仍用舊密鑰 | 影響這台設備能否重新接上當前帳號的雲端同步 |

## 備份時保存的數據密鑰

建立本地備份時，GranoFlow 會在備份開始前顯示「數據密鑰」。它預設隱藏，點眼睛圖標才顯示。你需要先把它抄錄並妥善保存，再勾選確認，才能開始建立備份。

這把數據密鑰對應當前和之後用它建立的備份。重新產生新數據密鑰，不會令舊備份改用新密鑰。也就是說，舊備份仍然需要舊密鑰。保存備份文件時，請同時保存當時的數據密鑰。

恢復備份時，GranoFlow 會先嘗試用當前設備已保存的密鑰打開備份。能打開時，不會額外要求你輸入；打不開時，才會要求你輸入建立這份備份時抄錄的數據密鑰。這裏要用的是「這份備份當時的密鑰」，不是隨手新產生的一把，也不是登入密碼。

## 首次同步時保存的數據密鑰

如果當前帳號雲端還沒有數據，GranoFlow 第一次把本機數據上傳到雲端前，也會要求你確認並保存數據密鑰。這時保存的密鑰會成為這套雲端同步數據的保護密鑰。以後新設備加入這套雲端同步，或某台設備需要雲端同步恢復時，通常就要輸入這把密鑰。

所以，首次同步時建議抄錄的密鑰和備份時建議抄錄的密鑰可能是同一把，也可能不是：

- 如果你沒有更改過數據密鑰，同一串密鑰可能同時打開這段時間建立的備份和當前雲端同步數據。
- 如果你在某次備份前或首次同步前重置過數據密鑰，新的備份或新的雲端同步保護會使用新密鑰，舊備份仍需要舊密鑰。
- 如果雲端早已有數據，當前設備本地顯示的新數據密鑰不能代替舊雲端同步密碼。

## 雲端同步甚麼時候要求輸入密鑰

當新設備加入已有雲端同步，或當前設備和雲端同步數據的密鑰不一致時，GranoFlow 會進入「雲端同步恢復」，並要求輸入「雲端同步密碼」。它用於嘗試打開當前帳號雲端已有的加密同步數據。

輸入正確後，GranoFlow 還會繼續判斷本機數據和雲端數據是否屬於同一套數據：

- 同一套數據：只更新這台設備的同步保護設定。
- 當前帳號雲端已有同步數據：以當前帳號雲端同步數據為準，把這台設備上可讀取的本機內容帶入當前帳號。
- 輸入不正確：不保存輸入，不下載、不上傳、不清空。

日常同步通常不需要反覆輸入密鑰。只有設備缺少當前雲端同步保護資訊、密鑰不匹配，或你正在讓新設備加入已有雲端數據時，才會要求你輸入。

## 改過數據密鑰後要記住甚麼

修改數據密鑰實際是在更新這套數據的加密保護入口。它會影響之後建立的備份，也會在會員帳號可同步時更新雲端同步保護。它不會重新加密歷史備份，也不會令舊備份突然可以用新密鑰打開。

最穩妥的保存方式，是按時間和用途保留記錄，不要只保留「最新密鑰」四個字。例如：

- `GranoFlow 雲端同步密碼，2026-06-22 後使用`
- `GranoFlow 本地備份 2026-06-18 數據密鑰`
- `舊電腦遷移前備份，可能使用舊密鑰`

## 應該怎樣保存

建議把密鑰保存到你離開當前設備後仍能訪問的地方，例如密碼管理器、安全筆記、離線紙本記錄或加密存儲。不要只保存截圖，也不要只存在 GranoFlow 裏，因為需要密鑰時，往往正是你換了設備、重裝系統，或舊環境打不開的時候。

一個簡單做法是給每條密鑰寫清楚用途：

- `GranoFlow 本地備份 2026-06-18 數據密鑰`
- `GranoFlow 雲端同步密碼`
- `舊電腦遷移前備份`

這樣以後恢復備份或恢復同步時，你能快速判斷該用哪一把。

## 在新設備上甚麼時候需要密鑰

以下情況可能需要你輸入舊設備或舊備份對應的密鑰：

- 換了手機或電腦
- 重裝了 GranoFlow
- 恢復 `.flow.grano` 本地備份
- 看到「雲端同步恢復」或雲端同步密碼輸入提示

輸入正確密鑰後，新設備才有機會訪問對應的備份或雲端加密數據。

## 輸入密鑰後會發生甚麼

GranoFlow 會先檢查這把密鑰能不能打開目標數據。恢復備份時，目標是備份文件；雲端同步恢復時，目標是當前帳號的雲端同步數據。

- **備份密鑰匹配** → 繼續恢復這份備份。
- **雲端同步密碼匹配，且本機和雲端屬於同一套數據** → 更新這台設備的同步保護設定，然後繼續同步。
- **雲端同步密碼匹配，但當前帳號雲端已有數據** → 以雲端數據為準，把這台設備上可讀取的本機內容帶入當前帳號的同步關係。
- **密鑰不對** → 不改變任何數據，讓你重新輸入。

## 忘記密鑰怎麼辦

按這個順序檢查：

1. **舊設備還能用嗎？** → 在舊設備上打開 GranoFlow，找到密鑰並複製
2. **密碼管理器裏有嗎？** → 檢查你常用的密碼管理器
3. **備份文件旁邊有備註嗎？** → 檢查備份目錄、網盤備註、安全筆記或遷移清單
4. **頁面有提供「檢查這台設備」一類修復入口嗎？** → 只有在這台設備仍有可用本地保護資訊時，它才可能幫你確認雲端關係
5. **舊設備還在，但 App 打不開？** → 聯絡 GranoFlow 支援，說明舊設備和當前情況

如果以上都沒有，雲端加密數據可能無法恢復。你仍可以檢查自己是否保存過 `.flow.grano` 本地備份和它對應的數據密鑰。

## 看到「輸入雲端同步密碼」時怎麼辦

如果 GranoFlow 提示你輸入雲端同步密碼，請填入當前雲端數據對應的完整密鑰。不要先清空雲端，也不要隨便輸入新產生的本機數據密鑰來試。恢復本地備份時保存的數據密鑰，只有在它和當前雲端同步保護本來就是同一把時，才能用於這裏。

輸入正確後，GranoFlow 會先判斷本機數據和雲端數據是不是同一份：

- 如果是同一份數據，只會更新這台設備的同步密鑰設定。
- 如果當前帳號雲端已有同步數據，會以當前帳號雲端數據為準，把這台設備上可讀取的本機內容帶入當前帳號。你只需要確認當前登入的是你要繼續使用的帳號。

:::caution[密鑰不是密碼，不能重置]
數據密鑰和雲端同步密碼丟失後，GranoFlow 無法幫你重置或找回已被它們保護的數據。現在就保存好，不要等到換設備或恢復備份時才發現缺了一把鑰匙。
:::
