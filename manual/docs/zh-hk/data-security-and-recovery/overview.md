如果你想知道 GranoFlow 點樣保護你的數據，記住一句：本地儲存、雲端同步、端對端加密、手動備份各自負責一件事。佢哋可以一齊降低風險，但任何一層都唔可以取代其他層。

GranoFlow 的數據保護體系有四層：

```text
本地儲存 → 雲端同步 → 端對端加密 → 手動備份
```

呢四層**互相配合，但不能互相替代**。手機遺失、App 重新安裝、換設備、誤刪重要任務時，你要先判斷問題屬於邊一層，再去對應位置處理。

## 四層各自負責什麼

| 層級 | 負責 | 不負責 |
| --- | --- | --- |
| **本地儲存** | 讓 App 離線時都可以使用本機數據 | 不防止設備損壞或遺失 |
| **雲端同步** | 讓多部設備之間共享同一份數據 | 不是備份；你誤刪之後，雲端通常都會跟住刪除 |
| **端對端加密** | 保護雲端數據不被第三方讀取 | 如果密鑰遺失或不匹配，你自己都打不開雲端數據 |
| **手動備份** | 在誤刪、遷移時提供最後一道防線 | 不是即時備份；備份只包含你建立備份當刻的數據 |

## 最常見的誤區

- **「我有雲端同步，所以數據安全」**：雲端同步會跟隨你的操作。你刪除了數據，雲端通常都會同步刪除。
- **「登出會清空數據」**：登出只係斷開帳號連接，本機數據會保留在目前設備上。
- **「刪除帳號等於清除所有數據」**：帳號刪除的是雲端數據。每部設備上的本地副本不會自動刪除。

## 遇到問題先由呢度判斷

```text
數據仲喺設備上？ → 去本地恢復或匯出
數據只喺雲端？ → 去新設備同步或下載
擔心數據外洩？ → 去加密設定
擔心誤刪？ → 去備份頁面
```

:::tip[最重要的一步]
保存你的加密恢復密鑰。呢個係密鑰不匹配時，唯一可以幫你找回雲端數據的憑證。
:::
