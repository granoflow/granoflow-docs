title: "管理里程碑"
description: "里程碑係項目裏面嘅階段檢查點，幫你將一個大目標拆成可以判斷進度嘅幾段。"
---

項目變大之後，最易令人焦慮嘅唔係任務好多，而係你唔知自己到底推進到邊步。任務話俾你聽「接下來做咩」，里程碑話俾你聽「呢段做到咩程度先算過關」。

所以，喺 GranoFlow 裏面，里程碑唔止係將任務分組嘅資料夾。佢更加似項目裏面嘅階段檢查點：一個里程碑下面可以放多項任務，但呢個階段真正要回答嘅係「呢個結果有冇達成」。當你咁樣理解里程碑時，項目頁面就唔再係一條好長嘅待辦清單，而係一張睇得出階段進度嘅路線圖。

假設你喺度準備一次課程發布。你可以將項目拆成「課程大綱確定」「錄製完成」「上線檢查」幾個里程碑。
每個里程碑下面放對應任務，例如寫大綱、錄製單元、檢查連結、準備發布文案。咁樣你每次打開項目時，見到嘅唔係「仲有 27 個任務」，而係「而家卡喺錄製完成之前，下一步要將呢段收咗佢」。

## 先諗清楚階段結果

建立里程碑前，先問一個簡單問題：呢個階段完成之後，人哋可以見到咩結果？

好嘅里程碑名稱通常似一個已經交付嘅結果，例如「初稿完成」「測試通過」「課程上線」。佢唔一定要好長，但最好可以令你過幾日返嚟時仍然睇得明佢嘅意思。

盡量少用「第一階段」「第二階段」呢類名稱。佢哋啱啱建立時睇落好整齊，但過咗一段時間之後，你仲要重新回憶第一階段究竟係咩。里程碑本來就係為咗減少呢種回憶成本。

## 添加里程碑

進入項目詳情頁後，撳「添加里程碑」，輸入名稱，再撳「添加」。新里程碑建立後，會出現喺項目裏面，你就可以將相關任務放到佢下面。

新增里程碑呢一步需要撳「添加」，因為佢係建立一個新嘅階段節點。建立完成之後，再編輯呢個已有里程碑時，儲存方式會變得唔同：多數修改會喺你完成對應動作後自動生效，唔需要再撳一個額外嘅儲存掣。

## 編輯已有里程碑時，幾時會儲存

已有里程碑嘅編輯採用自動儲存。你改完某個欄位、完成對應動作後，GranoFlow 會將修改寫入項目，唔需要喺頁面底部再搵「儲存」掣。

唔同內容嘅生效時機有少少分別：

| 你修改嘅內容 | 幾時生效 |
| --- | --- |
| 標題 | 修改後離開標題輸入框時儲存 |
| 截止日期 | 揀咗日期後儲存 |
| 描述 | 喺描述編輯頁儲存後生效 |
| 圖片 | 上傳或刪除圖片後生效 |
| PDF 附件 | 上傳或刪除 PDF 後生效 |

呢度容易誤解嘅一點係：自動儲存唔表示你仲未完成輸入時就一定已經提交。例如標題通常係喺你離開輸入框時儲存；描述會喺描述編輯頁裏面儲存後返返里程碑。你只需要照正常編輯節奏完成呢個動作，唔需要額外再撳一次總儲存。

如果你只係建立一個新里程碑，仍然要撳「添加」嚟完成建立。自動儲存講嘅係「已有里程碑」嘅後續修改。

## 右側編輯頁點樣關閉

喺闊屏或桌面佈局裏面，撳里程碑後，右側會打開里程碑編輯頁。佢唔係一個一定要提交嘅彈窗，而係項目詳情嘅一塊側邊編輯區域。

想收起右側編輯頁時，撳左側項目詳情裏面嘅空白區域就得。咁樣唔會改變你嘅項目內容，只係將右側編輯頁收起，令你返返項目整體視圖。

如果你撳嘅唔係空白區域，而係左側嘅其他項目、里程碑、任務或操作掣，GranoFlow 會執行你撳嘅動作。例如你撳另一個里程碑，右側會切換到嗰個里程碑；你撳導航欄轉去其他頁面，當前里程碑編輯頁會一齊關閉。

喺窄屏或手機佈局裏面，里程碑編輯頁會似一個新頁面咁打開。呢個時候用頂部返回掣返返項目詳情頁。

## 用進度同狀態判斷呢個階段係咪收尾

里程碑下面嘅任務會影響呢個階段嘅進度：任務完成得越多，進度越接近收尾。
但進度同里程碑狀態唔係同一件事。進度幫你判斷「仲有幾多」，狀態用嚟表示呢個階段係進行中、已完成，定係已經歸檔。

| 狀態 | 含義 |
| --- | --- |
| 進行中 | 呢個階段仲喺度推進，或下面仲有未完成任務 |
| 已完成 | 呢個階段已經被標記為完成，項目完成時亦可能會一齊更新里程碑狀態 |
| 已歸檔 | 你歸檔後，佢會從當前視圖收起，下面已完成嘅任務亦會一齊歸檔 |

歸檔里程碑前，裏面用戶見到嘅任務需要先完成或歸檔。如果仲有未完成任務，GranoFlow 會阻止歸檔，提醒你先處理呢啲任務。

## 調整里程碑順序

里程碑會按照項目詳情頁裏面嘅排列順序顯示。你可以拖曳里程碑嚟調整順序。

通常建議按項目推進順序排列：先發生嘅階段放上面，最後交付嘅階段放下面。咁樣由上往下睇，就係項目從準備到交付嘅路線。只有當你嘅項目更適合按主題組織時，先將同類階段放埋一齊。

## 刪除前先確認佢係空嘅

只有空里程碑會顯示刪除入口。如果裏面仲有任務，刪除掣唔會出現。你需要先移動、完成、歸檔或刪除呢啲任務，再返嚟刪除呢個里程碑。

:::caution[刪除里程碑唔會自動刪除任務]
刪除里程碑唔會連帶刪除任務。先處理任務，再刪除空里程碑。
:::

理解咗里程碑之後，下一步通常係將任務放入合適嘅項目同階段。繼續閱讀「將任務連接到項目」，你會見到任務掛到項目、掛到里程碑，以及佢係咪仲會出現喺今日視圖裏面嘅分別。
