---

如果你只想知道點樣保護 GranoFlow 數據：記住三件事。數據會先存在你而家用緊的設備；開啟雲端同步之後，雲端保存的是加密副本；登入密碼只係證明帳號屬於你，真正解開數據的是加密密鑰。

## 本地優先是甚麼意思

本地優先的意思是：GranoFlow 會先把數據存在你的設備上。你記錄任務、整理項目、查看回顧時，不需要一直連線。

如果你沒有開啟雲端同步，數據主要就在目前設備和目前工作區裏。換設備或重新安裝之前，你需要自己確認備份和密鑰都準備好。

如果你開啟雲端同步，GranoFlow 才會把數據上傳到雲端。雲端的作用是幫你在多部設備之間同步，或者在換設備後恢復數據。

> 本地 = 你正在使用的工作區
> 雲端 = 多設備同步和恢復用的加密副本

## 端到端加密保護甚麼

端到端加密（E2EE）的意思是：數據離開你的設備之前，已經先被加密。雲端保存的是密文，GranoFlow 的服務器讀不到你的任務內容。

但端到端加密不是萬能保護。你正常使用 GranoFlow 時，App 需要在本地解密數據，才可以把內容顯示給你看。如果設備被惡意軟件控制，或者其他人可以解鎖你的設備，端到端加密也擋不住。

所以你仍然要做好基本保護：鎖好屏幕，妥善保存密鑰，不要隨便把私人內容複製到不可信的服務裏。

## 密鑰不是登入密碼

這是最容易混淆的地方。登入密碼或電郵驗證碼，只負責證明這個帳號屬於你；加密密鑰，才負責解開你的加密數據。

| | 用途 |
|--|------|
| **登入密碼 / 電郵驗證碼** | 證明帳號屬於你 |
| **加密密鑰** | 解開你的加密數據 |

所以，你可能可以正常登入帳號，但如果沒有正確密鑰，雲端數據或備份檔案裏的內容仍然打不開。

這不是 GranoFlow 故意為難你，而是端到端加密的基本界線：如果服務器可以隨時幫你解開數據，也就代表服務器本來就可以讀你的內容。

**密鑰保存建議：**

- 存入可靠的密碼管理器
- 不要只截圖存在一部設備裏
- 換設備前確認密鑰可用

## 忘記密鑰會怎樣

忘記密鑰後，結果取決於你手上還有甚麼。

- 舊設備仍然可以打開數據：你可以繼續在舊設備上使用，或重新建立同步
- 只有雲端加密數據，但沒有密鑰：服務器無法替你解開
- 有備份檔案，但沒有建立備份時對應的密鑰：備份也無法恢復

> 登入讓你回到帳號，密鑰讓你解開數據。這兩件事不能互相代替。

## 換設備前要做甚麼

換設備前，先在舊設備上確認這三件事：

1. 舊設備仍然可以打開 GranoFlow
2. 雲端同步狀態正常，或已經匯出最新備份
3. 你知道目前數據對應的密鑰

確認後，再在新設備上登入同一帳號，按提示輸入密鑰，然後恢復或同步數據。

:::caution[空設備不是數據來源]
新設備第一次同步時，不要把一部沒有數據的設備當成「新起點」同步到雲端。這可能會覆蓋你的舊數據。
:::
