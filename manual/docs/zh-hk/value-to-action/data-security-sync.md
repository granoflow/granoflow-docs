一旦事情牵涉到“数据不见了”“换设备”“同步失败”或“密钥忘了”，人很容易一下子紧张起来。
這很正常，因為這些问题不只是技术问题，它們還会直接碰到一种不安：

> 我的记录還在吗？
> 我還能把它找回来吗？

這页的目的，就是把几个最容易混淆的概念拆开说清楚。如果你只想先记住最重要的三件事，可以先记住下面這三句：

1. 你的数据首先保存在你正在使用的设备上。
2. 开启云同步後，云端保存的是加密副本，不是可直接读取的原文。
3. 登录密码只能证明账号是你的，真正解开数据的是加密密钥。

把這三句话分清楚，很多关于同步和恢復的焦虑都会小很多。

## 本地优先是什么意思

“本地优先”的意思是：GranoFlow 会先把数据存在你的设备上。
你记录任務、整理項目、查看回顧時，不需要一直联网。

如果你没有开启云同步，数据主要就在当前设备和当前工作区裡。你换设备、重装系統或清理环境之前，需要自己先确认：备份和密钥是否都准备好了。

如果你开启云同步，GranoFlow 才会把数据上传到云端。云端的作用，不是替代本地，而是帮助你在多台设备之间同步，或者在换设备後恢復数据。

你可以先這样理解：

> 本地 = 你正在使用的数据来源
> 云端 = 用来同步和恢復的加密副本

所以，看到“本地优先”時，不必把它理解成“没有云”。
更准确的理解是：

> 先在本地工作，再决定是否讓云端帮你同步和恢復。

## 静默同步會做什么

如果你在這台设备上开启“静默同步”，其他设备完成云同步後，這台设备可以收到一个轻量同步信号，并在不打断当前操作的情况下自动补齐数据。

這个信号只用于告诉 App“云端可能有新版本”，不包含任務正文、附件内容、密钥或营销内容。它不会显示系統通知，也不需要你为静默同步单独打開系統通知权限。

iOS 和部分 Android 设备可能会延迟处理后台信号，所以 App 也会在启动或回到前台時补一次检查。补同步发生在前台時，你可能只会看到云同步入口进入同步中的状态；页面不会自动跳转，正在编辑的内容也不会被覆盖。

## 端到端加密保护什么

端到端加密（E2EE）的意思是：数据离开你的设备之前，已经先被加密。云端保存的是密文，GranoFlow 的服务器读不到你的任務内容、項目记录或回顧文本。

這能保护什么？
它能保护的是：即使数据被同步到云端，服务器也不能直接读取你的内容。

但端到端加密不是万能防护。
当你在设备上正常使用 GranoFlow 時，App 需要在本地解密数据，才能把内容显示给你看。如果设备本身被恶意软件控制，或者别人能直接解锁你的设备，端到端加密也挡不住。

所以，除了加密本身，你仍然需要做好這些基本保护：

- 锁好屏幕
- 妥善保存密钥
- 不要随便把私人内容复制到不可信的服务裡

加密解决的是“云端看不到原文”的问题，
但设备安全，仍然是你自己要守住的边界。

## 密钥不是登录密码

這是最容易混淆，也最容易讓人慌的一點。

登录密码或邮箱验证码，只负责证明這个账号属于你；
加密密钥，才负责解开你的加密数据。

<!-- markdownlint-disable MD060 -->
|  | 用途 |
| -- | ------ |
| **登录密码 / 邮箱验证码** | 证明账号属于你 |
| **加密密钥** | 解开你的加密数据 |
<!-- markdownlint-enable MD060 -->

所以，你可能能正常登录账号，但如果没有正确密钥，云端数据或备份文件裡的内容仍然打不开。

這不是 GranoFlow 故意為难你，而是端到端加密的基本边界：
如果服务器可以随時帮你解开数据，也就代表服务器本来就有能力读取你的内容。

换句话说：

> 登录讓你回到账号，密钥讓你回到内容。
> 這两件事不能互相替代。

**密钥保存建议：**

- 存进可靠的密码管理器
- 不要只截图存在一台设备裡
- 换设备前确认密钥可用

## 忘记密钥会怎样

忘记密钥後，结果取决于你手上還剩什么。

- **旧设备還能打開数据**：你可以繼續在旧设备上使用，或重新建立同步
- **只有云端加密数据，但没有密钥**：服务器无法替你解开
- **有备份文件，但没有创建备份時对應的密钥**：备份也无法恢復

很多人会希望“至少我已经登录账号了，系統能不能帮我找回来”。
但如果系統能在没有密钥的情况下帮你解开数据，那就意味着它本来就能读你的内容。那会直接破坏端到端加密的边界。

所以，與其把希望放在“之後再想办法”，不如在現在就把密钥保管好。

## 换设备前，先确认三件事

换设备前，最重要的不是赶快登录新设备，而是先在旧设备上确认：

1. 旧设备仍然能打開 GranoFlow
2. 云同步状态正常，或已经导出了最新备份
3. 你知道当前数据对應的密钥

确认後，再在新设备上登录同一账号，按提示输入密钥，然後恢復或同步数据。

如果這三件事裡有任何一件没确认，先不要着急切换。
先在旧设备上把边界搞清楚，通常比事後补救容易得多。

:::caution[空设备不是数据来源]
新设备第一次同步時，不要把一台没有数据的设备当成“新起點”同步到云端。這可能会覆盖你的旧数据。
:::

## 如果你現在正在慌，可以先這样判断

如果你遇到的是“登录了，但看不到内容”或“换设备後数据没出来”，先不要急着反复點按钮。先问自己：

1. 我現在看到的是本地没数据，還是云端数据解不开？
2. 我手上還有没有旧设备、备份文件或可用密钥？
3. 我現在缺的是账号登录，還是加密密钥？

把问题拆开後，很多看上去像同一种故障的情况，其实会变得清楚很多。
