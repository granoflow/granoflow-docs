你現在最需要做的是：在還能打開 GranoFlow 的裝置上，把關鍵金鑰儲存到密碼管理器或安全的地方。以後換裝置、重裝 App、還原本機備份，或看到「雲端同步還原」時，GranoFlow 可能會要求你輸入對應金鑰；沒有它，某些加密資料可能打不開。

![加密與還原金鑰 介面截圖](../../screenshots/zh-tw/data-encryption-recovery-key.png)

GranoFlow 的備份和雲端同步都使用加密。金鑰就像保險箱鑰匙：**沒有正確的鑰匙，連 GranoFlow 自己的伺服器也無法讀取受保護的資料**。這也代表：**如果你自己遺失金鑰，GranoFlow 無法幫你重設或找回已經被它保護的資料。**

你會看到兩個使用者介面名稱：建立本機備份或首次上傳雲端同步前，畫面顯示的是「資料密鑰」；雲端同步還原裡輸入的是「雲端同步密碼」。它們底層都服務於同一種加密保護，但名稱會跟著使用目標變化：打開備份時叫資料密鑰，打開目前帳號雲端同步資料時叫雲端同步密碼。

如果你從未更改過密鑰，同一串 `GF1-...` 可能既是某次備份的資料密鑰，也是目前雲端同步密碼。為了日後不混淆，儲存時仍建議寫清楚它來自哪一次備份、哪一次首次同步，或哪一次重設。

## 先分清三個名稱

<!-- markdownlint-disable MD060 -->
|  | 登入密碼（或驗證電子郵件） | 資料密鑰 | 雲端同步密碼 |
| --- | --- | --- | --- |
| 用來做什麼 | 證明你是誰 | 打開本機備份，或在首次同步時成為雲端保護密鑰 | 打開目前帳號已有的雲端同步資料 |
| 什麼時候看到 | 登入帳號時 | 建立本機備份、首次把本機資料同步到雲端，或修改資料密鑰時 | 新裝置加入既有雲端同步，或本機密鑰和雲端不匹配時 |
| 忘記了怎麼辦 | 可以重新發送驗證電子郵件 | **無法找回** | **無法找回** |
| 更改後會怎樣 | 只影響登入 | 之後建立的備份和之後的雲端保護設定會用新密鑰；舊備份仍用舊密鑰 | 影響這台裝置能否重新接上目前帳號的雲端同步 |

## 備份時儲存的資料密鑰

建立本機備份時，GranoFlow 會在備份開始前顯示「資料密鑰」。它預設隱藏，點眼睛圖示才顯示。你需要先把它抄錄並妥善儲存，再勾選確認，才能開始建立備份。

這把資料密鑰對應目前和之後用它建立的備份。重新產生新資料密鑰，不會讓舊備份改用新密鑰。也就是說，舊備份仍然需要舊密鑰。儲存備份檔案時，請同時儲存當時的資料密鑰。

還原備份時，GranoFlow 會先嘗試用目前裝置已儲存的密鑰打開備份。能打開時，不會額外要求你輸入；打不開時，才會要求你輸入建立這份備份時抄錄的資料密鑰。這裡要用的是「這份備份當時的密鑰」，不是隨手新產生的一把，也不是登入密碼。

## 首次同步時儲存的資料密鑰

如果目前帳號雲端還沒有資料，GranoFlow 第一次把本機資料上傳到雲端前，也會要求你確認並儲存資料密鑰。這時儲存的密鑰會成為這套雲端同步資料的保護密鑰。以後新裝置加入這套雲端同步，或某台裝置需要雲端同步還原時，通常就要輸入這把密鑰。

所以，首次同步時建議抄錄的密鑰和備份時建議抄錄的密鑰可能是同一把，也可能不是：

- 如果你沒有更改過資料密鑰，同一串密鑰可能同時打開這段時間建立的備份和目前雲端同步資料。
- 如果你在某次備份前或首次同步前重設過資料密鑰，新的備份或新的雲端同步保護會使用新密鑰，舊備份仍需要舊密鑰。
- 如果雲端早已有資料，目前裝置本機顯示的新資料密鑰不能代替舊雲端同步密碼。

## 雲端同步什麼時候要求輸入密鑰

當新裝置加入既有雲端同步，或目前裝置和雲端同步資料的密鑰不一致時，GranoFlow 會進入「雲端同步還原」，並要求輸入「雲端同步密碼」。它用於嘗試打開目前帳號雲端已有的加密同步資料。

輸入正確後，GranoFlow 還會繼續判斷本機資料和雲端資料是否屬於同一套資料：

- 同一套資料：只更新這台裝置的同步保護設定。
- 目前帳號雲端已有同步資料：以目前帳號雲端同步資料為準，把這台裝置上可讀取的本機內容帶入目前帳號。
- 輸入不正確：不儲存輸入，不下載、不上傳、不清空。

日常同步通常不需要反覆輸入密鑰。只有裝置缺少目前雲端同步保護資訊、密鑰不匹配，或你正在讓新裝置加入既有雲端資料時，才會要求你輸入。

## 改過資料密鑰後要記住什麼

修改資料密鑰實際是在更新這套資料的加密保護入口。它會影響之後建立的備份，也會在會員帳號可同步時更新雲端同步保護。它不會重新加密歷史備份，也不會讓舊備份突然可以用新密鑰打開。

最穩妥的儲存方式，是按時間和用途保留記錄，不要只保留「最新密鑰」四個字。例如：

- `GranoFlow 雲端同步密碼，2026-06-22 後使用`
- `GranoFlow 本機備份 2026-06-18 資料密鑰`
- `舊電腦遷移前備份，可能使用舊密鑰`

## 應該怎麼儲存

建議把金鑰儲存到你離開目前裝置後仍能存取的地方，例如密碼管理器、安全筆記、離線紙本記錄或加密儲存。不要只儲存截圖，也不要只存在 GranoFlow 裡，因為需要金鑰時，往往正是你換了裝置、重裝系統，或舊環境打不開的時候。

一個簡單做法是給每條金鑰寫清楚用途：

- `GranoFlow 本機備份 2026-06-18 資料密鑰`
- `GranoFlow 雲端同步密碼`
- `舊電腦遷移前備份`

這樣以後還原備份或還原同步時，你能快速判斷該用哪一把。

## 在新裝置上什麼時候需要金鑰

以下情況可能需要你輸入舊裝置或舊備份對應的金鑰：

- 換了手機或電腦
- 重裝了 GranoFlow
- 還原 `.flow.grano` 本機備份
- 看到「雲端同步還原」或雲端同步密碼輸入提示

輸入正確金鑰後，新裝置才有機會存取對應的備份或雲端加密資料。

## 輸入金鑰後會發生什麼

GranoFlow 會先檢查這把金鑰能不能打開目標資料。還原備份時，目標是備份檔案；雲端同步還原時，目標是目前帳號的雲端同步資料。

- **備份金鑰匹配** → 繼續還原這份備份。
- **雲端同步密碼匹配，且本機和雲端屬於同一套資料** → 更新這台裝置的同步保護設定，然後繼續同步。
- **雲端同步密碼匹配，但目前帳號雲端已有資料** → 以雲端資料為準，把這台裝置上可讀取的本機內容帶入目前帳號的同步關係。
- **金鑰不對** → 不改變任何資料，讓你重新輸入。

## 忘記金鑰怎麼辦

按這個順序檢查：

1. **舊裝置還能用嗎？** → 在舊裝置上打開 GranoFlow，找到金鑰並複製
2. **密碼管理器裡有嗎？** → 檢查你常用的密碼管理器
3. **備份檔案旁邊有備註嗎？** → 檢查備份目錄、雲端硬碟備註、安全筆記或遷移清單
4. **頁面有提供「檢查這台裝置」一類修復入口嗎？** → 只有在這台裝置仍有可用本機保護資訊時，它才可能幫你確認雲端關係
5. **舊裝置還在，但 App 打不開？** → 聯絡 GranoFlow 支援，說明舊裝置和目前情況

如果以上都沒有，雲端加密資料可能無法還原。你仍可以檢查自己是否儲存過 `.flow.grano` 本機備份和它對應的資料密鑰。

## 看到「輸入雲端同步密碼」時怎麼辦

如果 GranoFlow 提示你輸入雲端同步密碼，請填入目前雲端資料對應的完整密鑰。不要先清空雲端，也不要隨便輸入新產生的本機資料密鑰來試。還原本機備份時儲存的資料密鑰，只有在它和目前雲端同步保護本來就是同一把時，才能用於這裡。

輸入正確後，GranoFlow 會先判斷本機資料和雲端資料是不是同一份：

- 如果是同一份資料，只會更新這台裝置的同步密鑰設定。
- 如果目前帳號雲端已有同步資料，會以目前帳號雲端資料為準，把這台裝置上可讀取的本機內容帶入目前帳號。你只需要確認目前登入的是你要繼續使用的帳號。

:::caution[金鑰不是密碼，不能重設]
資料密鑰和雲端同步密碼遺失後，GranoFlow 無法幫你重設或找回已被它們保護的資料。現在就儲存好，不要等到換裝置或還原備份時才發現缺了一把鑰匙。
:::
