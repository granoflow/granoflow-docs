手機遺失、App 重新安裝、換設備、或者不小心刪掉重要任務——在這些時刻，你才會真正意識到「資料安全」有多重要。

GranoFlow 的資料保護體系有四層：

```
本機存儲 → 雲端同步 → 端對端加密 → 手動備份
```

這四層**互相配合，但不能互相替代**。

![資料與安全設定入口](../../../screenshots/zh-tw/interface-settings-account-data-entrypoints.png)

## 四層各自負責什麼

| 層級 | 負責 | 不負責 |
|------|------|--------|
| **本機存儲** | 讓 App 離線也能用 | 不防止設備損壞遺失 |
| **雲端同步** | 跨設備共享資料 | 不是備份，誤刪後雲端也會刪 |
| **端對端加密** | 保護雲端資料不被第三方讀取 | 遺失金鑰自己也打不開 |
| **手動備份** | 誤刪、遷移時的最後防線 | 不是即時的，有時效性 |

## 最常見的誤解

- **「我有雲端同步，所以資料安全」**：雲端同步會跟隨你的操作——你刪了，雲端也刪了。
- **「登出會清空資料」**：登出只斷開帳號連線，本機資料完整保留。
- **「刪帳號等於清除所有資料」**：帳號刪除的是雲端資料，每台設備上的本機副本不會自動刪除。

## 遇到問題先從這裏判斷

```
資料還在設備上？ → 去本機恢復或匯出
資料只在雲端？ → 去新設備同步或下載
擔心資料外洩？ → 去加密設定
擔心誤刪？ → 去備份頁面
```

:::tip[最重要的一步]
儲存你的加密恢復金鑰。這是唯一能讓你在金鑰不符時找回雲端資料的憑證。
:::
