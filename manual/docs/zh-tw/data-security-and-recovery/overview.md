如果你想知道 GranoFlow 怎麼保護你的資料，先記住一句話：本機儲存、雲端同步、端對端加密、手動備份各自負責一件事。它們會一起降低風險，但任何一層都不能取代其他層。

GranoFlow 的資料保護體系有四層：

```text
本機儲存 → 雲端同步 → 端對端加密 → 手動備份
```

這四層**互相配合，但不能互相替代**。手機遺失、App 重新安裝、換設備、誤刪重要任務時，你需要先判斷問題屬於哪一層，再到對應的位置處理。

## 四層各自負責什麼

| 層級 | 負責 | 不負責 |
| --- | --- | --- |
| **本機儲存** | 讓 App 離線時也能使用本機資料 | 不防止設備損壞或遺失 |
| **雲端同步** | 讓多台設備之間共享同一份資料 | 不是備份；你誤刪之後，雲端通常也會跟著刪除 |
| **端對端加密** | 保護雲端資料不被第三方讀取 | 如果金鑰遺失或不相符，你自己也打不開雲端資料 |
| **手動備份** | 在誤刪、遷移時提供最後一道防線 | 不是即時備份；備份只包含你建立備份當下的資料 |

## 最常見的誤解

- **「我有雲端同步，所以資料安全」**：雲端同步會跟隨你的操作。你刪除了資料，雲端通常也會同步刪除。
- **「登出會清空資料」**：登出只是中斷帳號連線，本機資料會保留在目前設備上。
- **「刪除帳號等於清除所有資料」**：帳號刪除的是雲端資料。每台設備上的本機副本不會自動刪除。

## 遇到問題先從這裡判斷

```text
資料還在設備上？ → 去本機恢復或匯出
資料只在雲端？ → 去新設備同步或下載
擔心資料外洩？ → 去加密設定
擔心誤刪？ → 去備份頁面
```

:::tip[最重要的一步]
儲存你的加密恢復金鑰。這是金鑰不相符時，唯一能幫你找回雲端資料的憑證。
:::
