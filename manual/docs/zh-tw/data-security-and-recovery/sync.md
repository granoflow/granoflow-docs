多端同步的作用是：你在一台裝置上新增、修改或刪除內容後，其他登入同一帳號的裝置也會盡量變成同樣的狀態。它讓你可以用手機、電腦和平板之間繼續同一套任務、專案、回顧和卡片工作流程，但它不是用來找回誤刪內容的備份功能。

把同步理解成「讓當前狀態在多台裝置之間收斂」會更準確。它關心的是現在應該一致，而不是三天前是什麼樣。真正需要保留歷史時間點時，請先建立 [本機備份](/manual/zh-tw/data-security-and-recovery/backup-and-restore/)。

## 同步什麼、不同步什麼

✅ 會同步的內容：

- 任務，例如標題、截止日期、標籤、狀態等
- 專案和里程碑
- 回顧記錄
- 圖片和附件，在網路允許的情況下同步
- 複習卡片相關的長期資產，在當前版本支援範圍內同步

⚠️ 需要特別記住：同步不是備份。

- **你刪了，其他裝置也會刪**：同步是雙向的，不會把刪除操作只留在一台裝置上。
- **沒有版本歷史**：同步不會保存「3 天前是什麼樣」。
- **圖片可能晚一點出現**：文字內容可能先同步完成，圖片和附件可能稍後才同步完成。
- **裝置偏好不等於帳號資料**：語言、主題、字型、部分本機設定和臨時快取通常屬於當前裝置，不應理解成所有裝置必須一致。

## 同步的常見狀態

| 狀態 | 含義 |
| --- | --- |
| 同步中 | 正在上傳或下載變更。 |
| 已同步 | 當前裝置的資料和雲端一致。 |
| 等待中 | 有變更正在排隊，通常和網路有關。 |
| 錯誤 | 同步遇到問題，需要檢查帳號或金鑰。 |

## 離線和服務不可達時會怎樣

如果暫時沒有網路，GranoFlow 裡的本機資料仍然可以使用。你可以繼續記錄任務、整理專案、寫回顧、搜尋內容，也可以匯出本機備份。

如果你還沒有登入、不是會員，或者當前無法連接同步服務，雲端同步會暫時不可用。你點擊需要聯網的功能時，App 會提示稍後重試；本機已有資料仍然可以繼續使用。

如果裝置有網路，但同步服務暫時不可達，App 也不會阻止你使用本機已有的資料。已經在這台裝置上的內容仍然可以編輯；需要上傳到雲端或從雲端下載的變化，會等你稍後再次同步。

會員裝置不需要單獨開啟靜默同步。只要帳號具備同步資格、服務端可達、金鑰狀態正常且當前裝置處於可執行狀態，GranoFlow 會預設嘗試在背景補齊其他裝置的新資料；這個過程保持靜默，不會彈系統橫幅，不會播放聲音，也不會新增通知中心訊息。

## 新裝置加入同步

如果你換了新手機，或者重裝了 App，想接上原來的雲端資料，需要使用舊裝置上的**雲端同步金鑰**。

這把密鑰通常就是第一次把這套資料同步到雲端時抄錄的資料密鑰。如果你後來改過資料密鑰，以目前雲端同步資料對應的那一把為準；舊備份仍然需要各自建立時的資料密鑰。

詳細步驟見 → [新裝置同步已有雲端資料](/manual/zh-tw/data-security-and-recovery/new-device-sync/)

## 看到「雲端同步還原」時怎麼辦

當 GranoFlow 發現當前裝置的本機金鑰和雲端同步資料不匹配時，會進入「雲端同步還原」。這不是普通錯誤彈窗，而是在保護你避免用錯誤的資料覆蓋雲端或本機。

還原流程會要求你輸入「雲端同步密碼」。這個密碼用來嘗試打開目前帳號雲端資料的加密材料，不是登入密碼，也不一定等於某個舊備份的資料密鑰。輸入後，GranoFlow 會先做檢查：

- 如果雲端資料和本機資料使用的是同一套資料密鑰，只會更新這台裝置的同步設定，然後繼續同步。
- 如果當前帳號已有受保護的雲端資料，且它和本機資料不是同一套，GranoFlow 會以當前帳號的雲端保護為目標，把本機資料遷入這個帳號。
- 如果輸入不正確，GranoFlow 不會保存這個輸入，也不會下載雲端資料、上傳本機資料或清空雲端。

你也可能看到「暫不同步」或「清空雲端資料」一類動作。暫不同步只會關閉當前還原入口，不會把它記成永久忽略；清空雲端資料是高風險操作，會繼續要求確認和系統認證。遇到不確定情況時，先暫停，確認舊裝置、備份檔案和當前帳號的雲端同步密碼是否都在手邊，再繼續處理。

## 同步和備份應該怎麼配合

日常使用時，讓同步負責多裝置的當前一致性；在做大操作前，讓備份負責可回退副本。一個可複用的判斷是：

- 今天只是想讓電腦和手機看到同一批任務，用同步。
- 準備重裝 App、換裝置、清理大量專案或匯入大量內容，先備份。
- 看到同步金鑰或雲端還原提示，不要先清空雲端；先確認舊裝置是否還能打開、是否有本機備份、是否保存過雲端同步密碼。

:::caution[同步不替代備份]
請定期匯出本機備份。誤刪的任務不能靠同步還原，因為雲端和其他裝置也會跟著刪除。
:::
