如果你記得任務、任務節點、專案、里程碑、卡片、日回顧、週回顧或月回顧內容裡的一兩個詞，但忘了它在哪裡，就用搜尋來把它找出來並開啟。搜尋的作用是「找已有內容」，不是做完整檢查，也不是替你整理任務或專案。

## 從哪裡進入

從首頁或主介面的搜尋入口進入搜尋頁。開啟後，在輸入框裡輸入比較明確的關鍵詞，例如任務標題、任務節點標題、專案說明、里程碑摘要、卡片正反面內容或回顧記錄中的幾個連續字詞，然後查看下面出現的結果列表。

你也可以在搜尋詞裡加入結構化條件：

- 輸入 `#標籤`，只看帶有這個標籤的任務。標籤可以寫系統標籤展示名或已有自訂標籤名，例如 `#重要`、`#緊急`、`#Work-Learning`、`#家`。
- 輸入 `@專案`，只看屬於這個專案的任務。專案名可以寫完整名稱，也可以寫能唯一指向專案的部分名稱。
- 普通關鍵詞可以和結構化條件一起使用，例如 `方案 #重要 @產品優化`。

如果只輸入普通關鍵詞，關鍵詞太短時頁面會提示你繼續輸入。`#標籤` 或 `@專案` 這種結構化條件可以單獨搜尋，不需要先湊夠三個字。

如果沒有結果，只表示目前可搜尋範圍內沒有匹配項。也可能是標籤名或專案名沒有對應到已有物件。它不代表 GranoFlow 已經逐項檢查了所有附件內容、已刪除內容或未納入搜尋範圍的歷史資料。

## 結果如何使用

搜尋結果按類型分欄顯示，只顯示有結果的分欄。分欄順序是任務、專案、里程碑、卡片、日回顧、週回顧、月回顧。任務內節點命中時，不會單獨出現「節點」分欄，而是把所屬任務顯示為一條任務結果，並在任務結果下列出命中的節點。這樣你既能看到命中原因，也能立刻知道它屬於哪個任務。

任務結果裡會展示標題、更新時間和命中的文字。多個節點、任務描述或任務回顧同時命中時，會在同一條任務結果下顯示最多兩行的命中節選。任務標題本身直接命中時，結果只顯示任務標題和基礎資訊，不再展開描述、回顧或節點內容。

點開某條任務結果後，GranoFlow 會按這個任務目前所在的位置帶你過去。它可能在收集箱、任務列表、已完成、封存或回收桶裡。點開專案或里程碑結果會進入對應專案上下文；點開卡片結果會進入對應卡片詳情；點開回顧結果會進入成就與回顧頁面，並盡量定位到對應日期或週。

如果結果屬於某個專案，開啟後仍然要回到任務或專案頁面裡繼續判斷：它屬於哪個階段、和哪個里程碑有關、日期是否還合適。

搜尋可以幫你找回已有卡片，但不替代卡片管理、複習或封存判斷。想系統整理卡片時，先從 [卡片：把經驗帶回行動](/manual/zh-tw/review-cards/) 理解卡片和任務的關係，再進入卡片統計或卡片管理。

## 適合什麼時候用

- 你記得任務、任務節點、專案、里程碑、卡片、日回顧、週回顧或月回顧文字的一部分，但忘了它放在哪。
- 你想快速開啟一個已完成或已封存的任務。
- 你在整理收集箱、專案或做回顧前，想先找出某個舊任務。

搜尋不會建立新任務，不會建立新標籤或新專案，不會批次修改搜尋結果，也不會儲存成自動篩選檢視。如果你需要長期按標籤、專案、日期或完成狀態查看任務，請繼續使用對應的列表和專案頁面。
