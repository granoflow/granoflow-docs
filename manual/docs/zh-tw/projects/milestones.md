title: "管理里程碑"
description: "里程碑是專案裡的階段檢查點，幫你把一個大目標拆成可以判斷進度的幾段。"
---

專案變大以後，最容易讓人焦慮的不是任務很多，而是你不知道自己到底推進到哪一步了。任務告訴你「接下來做什麼」，里程碑告訴你「這一段做到什麼程度才算過關」。

所以，在 GranoFlow 裡，里程碑不只是把任務分組的資料夾。它更像專案裡的階段檢查點：一個里程碑下面可以放多項任務，但這個階段真正要回答的是「這個結果有沒有達成」。當你這樣理解里程碑時，專案頁面就不再是一條很長的待辦清單，而是一張能看出階段進度的路線圖。

假設你在準備一次課程發布。你可以把專案拆成「課程大綱確定」「錄製完成」「上線檢查」幾個里程碑。
每個里程碑下面放對應任務，例如寫大綱、錄製單元、檢查連結、準備發布文案。這樣你每次打開專案時，看到的不是「還有 27 個任務」，而是「現在卡在錄製完成之前，下一步要把這一段收掉」。

## 先想清楚階段結果

建立里程碑前，先問一個簡單問題：這個階段完成以後，別人能看見什麼結果？

好的里程碑名稱通常像一個已經交付的結果，比如「初稿完成」「測試通過」「課程上線」。它不一定很長，但最好能讓你過幾天回來時仍然看得懂它的意義。

盡量少用「第一階段」「第二階段」這種名稱。它們在剛建立時看起來很整齊，但過一段時間以後，你還要重新回想第一階段到底是什麼意思。里程碑本來就是為了減少這種回想成本。

## 新增里程碑

進入專案詳情頁後，點擊「新增里程碑」，輸入名稱，再點擊「新增」。新里程碑建立後，會出現在專案裡，你就可以把相關任務放到它下面。

新增里程碑這一步需要點「新增」，因為它是在建立一個新的階段節點。建立完成以後，再編輯這個既有里程碑時，儲存方式會變得不同：多數修改會在你完成對應動作後自動生效，不需要再點一個額外的儲存按鈕。

## 編輯既有里程碑時，什麼時候會儲存

既有里程碑的編輯採用自動儲存。你改完某個欄位、完成對應動作後，GranoFlow 會把修改寫入專案，不需要在頁面底部再找「儲存」按鈕。

不同內容的生效時機略有差異：

| 你修改的內容 | 什麼時候生效 |
| --- | --- |
| 標題 | 修改後離開標題輸入框時儲存 |
| 截止日期 | 選擇日期後儲存 |
| 描述 | 在描述編輯頁儲存後生效 |
| 圖片 | 上傳或刪除圖片後生效 |
| PDF 附件 | 上傳或刪除 PDF 後生效 |

這裡容易誤解的一點是：自動儲存並不表示你還沒完成輸入時就一定已經提交。比如標題通常是在你離開輸入框時儲存；描述會在描述編輯頁裡儲存後回到里程碑。你只需要按正常編輯節奏完成這個動作，不需要額外再點一次總儲存。

如果你只是建立一個新里程碑，仍然要點擊「新增」來完成建立。自動儲存說的是「既有里程碑」的後續修改。

## 右側編輯頁怎麼關閉

在寬螢幕或桌面佈局裡，點擊里程碑後，右側會打開里程碑編輯頁。它不是一個必須提交的彈窗，而是專案詳情的一塊側邊編輯區域。

想收起右側編輯頁時，點擊左側專案詳情裡的空白區域即可。這樣不會改變你的專案內容，只是把右側編輯頁收起來，讓你回到專案整體檢視。

如果你點擊的不是空白區域，而是左側的其他專案、里程碑、任務或操作按鈕，GranoFlow 會執行你點的動作。比如你點另一個里程碑，右側會切換到那個里程碑；你點導覽列切到其他頁面，目前里程碑編輯頁會一起關閉。

在窄螢幕或手機佈局裡，里程碑編輯頁會像一個新頁面一樣打開。此時使用頂部返回按鈕回到專案詳情頁。

## 用進度和狀態判斷這個階段是否收尾

里程碑下面的任務會影響這個階段的進度：任務完成得越多，進度越接近收尾。
但進度和里程碑狀態不是同一件事。進度幫你判斷「還剩多少」，狀態用來表示這個階段現在是進行中、已完成，還是已經封存。

| 狀態 | 含義 |
| --- | --- |
| 進行中 | 這個階段還在推進，或下面還有未完成任務 |
| 已完成 | 這個階段已經被標記為完成，專案完成時也可能一起更新里程碑狀態 |
| 已封存 | 你封存後，它會從目前檢視收起，下面已完成的任務也會一起封存 |

封存里程碑前，裡面使用者可見的任務需要先完成或封存。如果還有未完成任務，GranoFlow 會阻止封存，提醒你先處理這些任務。

## 調整里程碑順序

里程碑會按照專案詳情頁裡的排列順序顯示。你可以拖曳里程碑來調整順序。

通常建議按專案推進順序排列：先發生的階段放在上面，最後交付的階段放在下面。這樣從上往下看，就是專案從準備到交付的路線。只有當你的專案更適合按主題組織時，才把同類階段放在一起。

## 刪除前先確認它是空的

只有空里程碑會顯示刪除入口。如果裡面還有任務，刪除按鈕不會出現。你需要先移動、完成、封存或刪除這些任務，再回來刪除這個里程碑。

:::caution[刪除里程碑不會自動刪除任務]
刪除里程碑不會連帶刪除任務。先處理任務，再刪除空里程碑。
:::

理解了里程碑以後，下一步通常是把任務放進合適的專案和階段。繼續閱讀「把任務連接到專案」，你會看到任務掛到專案、掛到里程碑，以及它是否還會出現在今日檢視裡的差異。
