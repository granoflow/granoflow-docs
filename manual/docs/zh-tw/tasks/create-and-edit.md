新增任務的規則只有一條：**寫個標題，儲存**。其他欄位都是可選的，等你需要的時候再填。

## 從哪裡建立任務

| 入口 | 適合的場景 |
| --- | --- |
| 底部 **+** 按鈕 | 隨時隨地的快速錄入 |
| 收集箱頁面內的輸入框 | 專門整理收集箱時 |
| 專案或里程碑頁面內 | 直接歸屬到某個專案 |
| 已有任務詳情裡的節點 | 拆分一個大任務的步驟 |

## 任務編輯介面

![建立與編輯任務對話框](../../../screenshots/zh-tw/tasks-create-edit-dialog.png)

各欄位說明：

| 欄位 | 是否必填 | 作用 |
| --- | --- | --- |
| 標題 | ✅ 必填 | 任務的名稱，越具體越容易執行 |
| 描述 | 可選 | 背景資訊、連結、備注 |
| 截止日期 | 可選 | 設定後任務出現在對應日期的任務清單 |
| 提醒 | 可選 | 在指定時間發通知，不能設在過去 |
| 專案 | 可選 | 設定後任務從收集箱移入該專案 |
| 里程碑 | 可選 | 設定後歸屬到專案的某個階段 |
| 標籤 | 可選 | 方便篩選，可以有多個 |
| 節點 | 可選 | 把任務拆成更小的步驟 |

:::tip[善用自然語言輸入]
在標題輸入框裡，你可以直接寫 `#標籤名`、`@日期`、`~提醒時間`，GranoFlow 會自動解析。詳細的解析規則見[用自然語言寫任務](title-parser)。
:::

## 儲存後任務去哪了

這取決於你填了什麼：

- **沒有日期、沒有專案** → 進收集箱
- **有日期** → 出現在那一天的任務清單
- **有專案** → 出現在專案裡
- **在專案頁面裡建立** → 直接歸屬到那個專案

修改欄位不是新建一個任務，而是改變同一個任務的歸屬。

## 編輯已有任務

點擊任何任務即可打開詳情頁。修改你想改的欄位，離開時自動儲存。

:::caution[注意]
提醒不能設定在已經過去的時間。如果你設的提醒時間已經過了，系統會提示你重新選。
:::

完成、封存和刪除是三種不同操作，填寫欄位不會讓任務自動變成完成狀態。
