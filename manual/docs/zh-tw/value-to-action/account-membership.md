当会员没有显示、恢復购买没反應、换设备後功能没解锁時，最容易发生的一件事是：

> 所有词都混在一起了。

账号、订阅、会员、权益，听起来像是在说同一件事，但它們其实不是同一层东西。

如果這几个概念没有拆开，问题会很难定位；
如果拆开了，很多情况就会一下子清楚很多。

如果你只想先记住最重要的三件事，可以先记住：

1. **账号**决定“這些数据和权益属于谁”
2. **订阅**决定“你是否在 Apple 或 Google 上完成了购买關係”
3. **权益**决定“当前這个账号实际能用哪些功能”

所以，“我點过购买按钮”不一定等于“当前账号已经拿到权益”。
中间還隔着平台购买、购买状态整理、服务端同步和 App 刷新显示。

## 四个词，分别是什么意思

| 词 | 是什么 |
| --- | --- |
| **账号** | 你在 GranoFlow 的身份，用来识别数据和权益属于谁 |
| **订阅** | 你在 Apple 或 Google 平台上的购买關係 |
| **会员** | GranoFlow 对外描述的用户身份，例如 Pro |
| **权益** | 当前账号实际可以使用哪些功能 |

你可以把流程理解成這样：

```mermaid
graph LR
  A[平台购买] --> B[RevenueCat 整理购买状态]
  B --> C[GranoFlow 服务端同步权益]
  C --> D[App 展示可用功能]
```

這张图想说明的不是系統多复杂，
而是提醒你：

> 平台购买成功，只是链条中的一环。
> 只有每一环都对齐，App 裡看到的结果才会正确。

## 登录到底解决什么问题

不登录也可以使用 GranoFlow 的本地功能，例如记录任務、整理專案、写回顧。

登录账号後，GranoFlow 才能确认這些内容和权益属于谁。以下功能通常需要登录並经过服务端确认：

- 云同步
- 多设备使用
- 会员权益识别
- 恢復购买
- 账号删除

你可以把两者先简单分开：

> 本地使用解决“我現在怎么记录”。
> 登录账号解决“這些数据和权益属于谁”。

如果登录、购买服务或云同步暂時不可用，本地功能不会因此停止。只是登录、同步、权益确认和恢復购买需要稍後再试。

## 恢復购买是什么，不是什么

换设备或重装 App 後，如果会员权益没有出现，可以尝试“恢復购买”。
它的作用是：讓 Apple 或 Google 重新检查购买记录，再和当前登录的 GranoFlow 账号权益对齐。

恢復购买的重點，不是“重新买一次”，而是“重新确认這笔购买属于哪个账号、当前是否仍然有效”。

但恢復购买不能解决所有情况：

- 如果购买绑定的是另一个 GranoFlow 账号，当前账号不会自动得到权益
- 如果订阅已经退款或过期，恢復购买也不会重新开通权益
- 如果购买服务暂時不可用，App 会提示稍後重试，本地数据不会受到影响

所以，当恢復购买没有立刻解决问题時，不一定是功能坏了，
也可能只是链条裡的别的环节還没对上。

## 桌面端為什么没有购买按钮

桌面端，也就是 Windows、macOS 和 Linux 版本，為了符合各平台分发规则，可能不显示购买入口。

這不代表桌面端少了会员功能。
如果你已经有会员，在桌面端登录同一个 GranoFlow 账号後，对應功能会正常解锁。

需要购买会员時，请从 iOS 或 Android 端操作。

所以，桌面端没有购买按钮，並不等于桌面端不能使用会员权益。
它只是购买入口放在了更合适的平台上。

## 遇到问题時，先别急着重试很多次

如果你发现会员没有显示、恢復购买没有反應，先不要立刻反复點按钮。
更有效的方式是先回答下面几个问题：

1. 我現在登录的是哪个 GranoFlow 账号？
2. 购买发生在哪个平台，Apple 還是 Google？
3. 当前订阅是否還有效？
4. App 是否已经刷新了权益？
5. 有没有把不同账号搞混？

這几个问题能定位绝大多数会员和权益问题。

很多時候，问题不是出在“会员没了”，而是出在：

- 登录了另一个账号
- 购买平台和当前设备不一致
- 订阅已经失效
- 权益還没刷新出来

把這些环节分开以後，你会比单纯焦急地反复操作更快接近答案。

## 這页最想帮你分清的一件事

会员问题最讓人头大的地方，不是它技术上多复杂，而是它很容易讓人把“账号”“购买”“订阅”“权益”混成一团。

這页想帮你恢復的，只是一个清楚判断：

- 我現在的问题出在账号，還是出在购买？
- 我缺的是登录，還是权益同步？
- 我現在應该恢復购买，還是先确认自己登录的是谁？
