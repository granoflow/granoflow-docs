---

如果你只想知道怎麼保護 GranoFlow 資料：記住三件事。資料會先存在你正在使用的裝置上；開啟雲端同步後，雲端保存的是加密副本；登入密碼只能證明帳號是你的，真正解開資料的是加密金鑰。

## 本地優先是什麼意思

本地優先的意思是：GranoFlow 會先把資料存在你的裝置上。你記錄任務、整理專案、查看回顧時，不需要一直連網。

如果你沒有開啟雲端同步，資料主要就在目前裝置和目前工作區裡。換裝置或重新安裝前，你需要自己確認備份和金鑰都準備好了。

如果你開啟雲端同步，GranoFlow 才會把資料上傳到雲端。雲端的作用是幫你在多台裝置之間同步，或是在換裝置後恢復資料。

> 本地 = 你正在使用的工作區
> 雲端 = 多裝置同步和恢復用的加密副本

## 端對端加密保護什麼

端對端加密（E2EE）的意思是：資料離開你的裝置之前，已經先被加密。雲端保存的是密文，GranoFlow 的伺服器讀不到你的任務內容。

但端對端加密不是萬能保護。你正常使用 GranoFlow 時，App 需要在本地解密資料，才能把內容顯示給你看。如果裝置被惡意軟體控制，或是其他人可以解鎖你的裝置，端對端加密也擋不住。

所以你仍然需要做好基本保護：鎖好螢幕，妥善保存金鑰，不要隨便把私人內容複製到不可信的服務裡。

## 金鑰不是登入密碼

這是最容易混淆的地方。登入密碼或電子郵件驗證碼，只負責證明這個帳號屬於你；加密金鑰，才負責解開你的加密資料。

| | 用途 |
|--|------|
| **登入密碼 / 電子郵件驗證碼** | 證明帳號屬於你 |
| **加密金鑰** | 解開你的加密資料 |

所以，你可能可以正常登入帳號，但如果沒有正確金鑰，雲端資料或備份檔案裡的內容仍然打不開。

這不是 GranoFlow 故意為難你，而是端對端加密的基本界線：如果伺服器可以隨時幫你解開資料，也就代表伺服器本來就可以讀你的內容。

**金鑰保存建議：**

- 存進可靠的密碼管理器
- 不要只截圖存在一台裝置裡
- 換裝置前確認金鑰可用

## 忘記金鑰會怎樣

忘記金鑰後，結果取決於你手上還剩什麼。

- 舊裝置仍然可以打開資料：你可以繼續在舊裝置上使用，或重新建立同步
- 只有雲端加密資料，但沒有金鑰：伺服器無法替你解開
- 有備份檔案，但沒有建立備份時對應的金鑰：備份也無法恢復

> 登入讓你回到帳號，金鑰讓你解開資料。這兩件事不能互相替代。

## 換裝置前要做什麼

換裝置前，先在舊裝置上確認這三件事：

1. 舊裝置仍然可以打開 GranoFlow
2. 雲端同步狀態正常，或已經匯出最新備份
3. 你知道目前資料對應的金鑰

確認後，再在新裝置上登入同一帳號，按提示輸入金鑰，然後恢復或同步資料。

:::caution[空裝置不是資料來源]
新裝置第一次同步時，不要把一台沒有資料的裝置當成「新起點」同步到雲端。這可能會覆蓋你的舊資料。
:::
